## 2. 页面概述

本页分析楼盘与具体房源的供给规模、发布质量、可租面积、价格和属性结构，并在前台事件已接入时提供曝光与咨询归因。成功标准是租赁与出售口径不混用，任一分布可下钻至具体楼盘或房源，过期库存可定位责任主体。MVP 不建设推荐算法、广告效果归因、价格预测或交易成交金额分析。

## 3. 用户与权限

平台管理员须具备“数据看板-房源分析”菜单权限并可查看全量；运营人员须具备该菜单权限且仅查看授权城市；销售主管须具备该菜单权限且仅查看所属团队经纪人负责、并位于授权城市的楼盘和房源；经纪人、客服无本页菜单。聚合导出另需“房源分析导出”操作权限，查看 `LI-12`、`LI-13` 另需“前台事件分析”操作权限。城市、楼盘、商户、经纪人和标签均先按上述数据范围过滤；所有聚合 KPI、趋势点、分布桶和排行均只能先进入本页分析明细表，明细中的单对象行通过权限校验后才可打开业务详情。前台事件只可按聚合维度显示，禁止展示访问者手机号、设备标识、IP 或可识别轨迹。无前台事件分析权限时，曝光和咨询归因区不渲染。

## 4. 页面入口与关联

- 菜单入口：`数据看板 > 房源分析`；经营概览 OP-01、OP-02、OP-03 可带当前条件进入。
- 上游：楼盘、房源、审核、发布版本、商户/经纪人资格、媒体、标签、区域和可选前台 `listing_exposure`、`listing_inquiry` 事件。
- 下游：点击任一聚合 KPI、趋势点、分布桶或排行一律先进入本页分析明细表，并携带 `metric_id`、查询快照、解析 ID 集合、分子/分母集合角色、逐行证据和实际时间谓词；仅分析明细中的单对象行可再打开楼盘、房源、审核、举报、商户、经纪人或线索的对应业务详情。
- 返回：下钻后恢复时间范围、存量时点、分布桶、筛选和排序。

## 5. 页面结构

页面依次展示范围栏、供给与发布质量卡片、可用面积与价格分布、装修/类型/标签/位置分布、滞销库存队列及可选的前台曝光-咨询归因区。默认显示近 30 日流量和当前时点存量；所有存量卡片明确“截至”时间，分布图可切换数量、面积和占比。曝光归因区仅在事件数据源健康且用户有权限时出现，缺源时显示“不适用：未接入前台事件”，不显示 0。

## 6. 查询与筛选

| 条件 | 规则 |
|---|---|
| 时间范围 | 今日、近 7 日、近 30 日、自定义；默认近 30 日，最长 365 天，按北京时间。 |
| 存量时点 | 默认当前时刻；允许选择周期结束时刻，用于与对比期一致的存量比较。 |
| 城市/区域/商圈/楼盘 | 级联多选，只显示授权范围；停用位置可作为历史维度查询。 |
| 商户/经纪人 | 仅显示有权实体；按事件时归属或存量时当前归属的口径在图表提示中明确。 |
| 租售、楼盘类型、装修、标签、直租/直售 | 多选；租赁价格与出售价格分桶独立，切换“全部”时分别展示而不合并均价。 |
| `publication_status` / `review_status` | 发布状态仅可选草稿/已上架/已下架/已出租，审核状态仅可选未提交/待审核/审核通过/已驳回；默认全部，活跃类指标按第 12 节谓词执行。 |
| `review_task_status` | 仅审核队列指标使用，独立可选待领取/审核中/已退回/已完成/已撤回；不得写入或替代 Listing 的 `review_status`。 |
| 暂停、更新时间 | 支持质量和滞销排查；暂停是独立前台可见性约束，不属于发布或审核状态。 |

条件与图表度量写入 URL；清空恢复默认范围。面积和价格区间下限不得大于上限，价格必须携带币种、周期和单位。

## 7. 列表或内容展示

卡片使用 `LI-01` 至 `LI-06`；分布图使用 `LI-07` 至 `LI-10`，每个柱段显示数值、占比、统计对象和分桶说明。LI-07 的标准价格桶与“未标准化/未知价格”桶、LI-08 的有效成员与“未知”桶均互斥且覆盖其完整筛选集合；所有桶共用总分母，未四舍五入占比之和为 100%，展示取整后以末位校正保证合计 100%。价格分布默认租赁按“元/平方米/天”标准化的字典桶，出售按“元/平方米”或总价字典桶分别展示；原始币种或周期不可换算时进入“未标准化/未知价格”桶并可下钻。滞销队列按最后有效维护时间升序，显示房源、楼盘、商户、负责人、当前状态、天数和风险；前台归因区显示 `LI-12`、`LI-13` 与事件覆盖率，咨询类型仅展示咨询量分布，不展示按咨询类型切分的曝光咨询率。

## 8. 详情及表单

本页无新增或编辑表单。指标说明抽屉展示完整字典、状态范围、标准化版本、事件覆盖率、查询快照和下钻链接。第 12 节的每个指标均先进入本页分析明细表，不能仅依赖目标业务页的通用筛选：接口必须接收 `metric_id`、`query_snapshot_id`、服务端解析的 `resolved_building_ids`、`resolved_listing_ids`、`resolved_review_ids` 或 `resolved_event_ids`，以及本指标的精确时间谓词；表格逐行展示分子/分母命中、集合角色、`evidence_type`、`evidence_id`、`evidence_at`、状态/价格标准化快照和桶边界，再链接单条房源、审核任务或事件记录。任何聚合项不得直接进入业务房源列表；业务页链接仅打开分析明细中的单对象行，不把分析派生字段伪装成目标页筛选。滞销行可打开房源只读侧栏并跳转编辑、审核或举报处理，仍由原页面权限控制。导出支持当前聚合表或滞销明细；导出继承筛选、权限与脱敏规则，前台事件导出不得带访客识别字段。

## 9. 核心操作与流程

1. 用户选择时间、存量时点和维度，系统先校验权限与价格单位可比性。
2. 存量与流量指标从同一查询快照读取；价格标准化失败或缺价的记录进入“未标准化/未知价格”桶，属性缺失进入“未知”桶，均纳入 LI-07/LI-08 的总分母和下钻集合，不静默排除。
3. 用户切换数量/面积、分布维度或点击桶，先打开本页分析明细表；明细请求接收指标 ID、桶边界、状态、快照 ID、解析 ID 集合、分子/分母集合角色、逐行证据和精确时间谓词，单对象行才可再打开业务详情。
4. 若前台事件源已启用，系统按 `listing_id` 和事件版本关联；事件源不健康、未接入或覆盖不足时只显示状态说明，不能以缺失补 0。

查询、切换、下钻、导出、滞销侧栏和事件源状态均记录审计；本页不可修改发布状态或指标口径。

## 10. 状态与业务规则

Listing `publication_status` 仅为草稿/已上架/已下架/已出租，Listing `review_status` 仅为未提交/待审核/审核通过/已驳回；审核队列 `review_task_status` 仅为待领取/审核中/已退回/已完成/已撤回，三个字段独立存储、展示和统计。活跃房源和有效供给指标必须完整复用 README 统一有效供给谓词，包括审核/发布、`supply_visibility_hold=正常`、举报暂停、媒体完整、Building/城市/区域启用、当前商户关系及商户/资质/服务城市有效；租赁还要求可租且可用日期未结束，任何页面不得放宽。其历史事件保留且只作历史分析。一个同时租赁和出售的物理单元必须使用不同 `listing_id`，不得跨租售类型聚合价格或发布率。

## 11. 异常与边界

- 缺少面积、价格数值、币种、周期、单位、标签或位置的记录进入“未知/未标准化”桶，不静默丢弃；LI-07 的标准价格桶与“未标准化/未知价格”桶、LI-08 的有效成员与“未知”桶必须互斥覆盖同一完整筛选集合，所有桶纳入同一总分母且比例合计 100%，每个桶均可下钻到分子集合和完整分母集合。LI-10 的城市、区域、商圈或楼盘位置缺失、为空或不可解析时均归入“未知位置”治理桶，计入总量和占比分母，只有真正测试或无效记录才排除；对应质量计数可下钻。
- 楼盘停用、商户资格失效或举报暂停在分析期间发生时，流量按事件时可见性，存量按选择时点状态；两者不得相互替代。
- `last_effective_maintained_at` 的维护事件及推进条件以 `03_房源管理/01_房源列表_PRD.md` 的 Listing 契约为唯一权威；`updated_at` 或批量迁移时间不得替代该字段，也不构成有效维护。
- 未接入、延迟超过 30 分钟或覆盖率低于 95% 的前台事件区标明不可用范围，并通知具备“数据质量治理”操作权限的平台管理员及对应授权城市的运营人员；其他房源指标不受影响。
- 无权限、空数据、接口失败和并发刷新分别显示禁止、空状态、重试和最近成功快照。

## 15. 依赖与风险

依赖房源主数据、发布/审核/暂停状态、楼盘和商户资格、区域与标签字典、维护事件、前台埋点和权限服务。主要风险为历史价格单位不完整、可用日期缺失、标签停用、前台事件漏报和机器人流量；上线前需完成价格标准化策略、事件去重规则、事件覆盖率监控及与房源列表的抽样对账。前台事件未接入不阻塞核心供给分析。

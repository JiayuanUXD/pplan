# 楼盘列表 PRD

## 1. 文档信息
编号：02-01；版本：MVP v1.0；状态：可评审；所属模块：楼盘管理；负责人角色：运营人员；更新时间：2026-07-17。本文以楼盘为可被多个房源复用的主数据，不能把具体出租或出售单元录入为楼盘。

## 2. 页面概述
运营通过本页建立并维护前台楼盘列表、详情页和地图找房所需主数据。成功标准是启用且资料完整的楼盘可被选择创建房源，并在存在合格上架房源时向前台提供正确的图集、价格聚合、位置、顾问和配套信息。MVP 覆盖查询、创建、编辑、详情、前台预览、启停及导出；不建设广告投放或交易闭环。本页 UI 由中性线框或自有设计稿承接；参考截图仅用于竞品能力研究与需求追溯，不是产品界面、交付素材或视觉参考。

## 3. 用户与权限
平台管理员拥有全部城市的查看、新增、编辑、启停、合并、导入、导出和审计权限。运营人员仅可查看获授权城市，并分别取得新增编辑、启停、合并、导入、导出操作权限；没有对应操作权限时按钮不展示且接口拒绝。销售主管可只读查看所属团队经纪人负责且位于授权城市的楼盘；经纪人可只读查看本人负责楼盘；客服无菜单权限。顾问手机号仅平台管理员、该经纪人本人及其销售主管在字段权限允许时显示全量，其余可见角色按 `138****1234` 脱敏；查询、详情、统计和导出均先应用城市及团队数据范围。

## 4. 页面入口与关联
从“楼盘管理/楼盘列表”进入。新建前必须存在启用的城市、行政区和商圈，并选择有效商户作为默认供给方；编辑页从列表“编辑”或详情页进入。楼盘详情可跳转到按 `building_id` 过滤的房源列表、商圈节点、顾问档案与前台预览；房源新建只可选择启用楼盘。返回列表时保留本会话的筛选、页码和排序。

## 5. 页面结构
页面自上而下为操作栏（新增、导入、导出）、筛选区、结果统计与批量操作栏、分页表格。行内“名称”打开详情抽屉，抽屉分基础、位置与交通、楼宇参数、媒体、配套、顾问、关联房源、操作记录八组；“编辑”打开同字段表单。筛选区默认展开，表头与操作列固定；窄屏横向滚动表格，危险操作不折叠为无文字图标。

## 6. 查询与筛选
支持城市、行政区、商圈、地铁站、楼盘名称/别名、租售支持类型、楼盘类型/等级、装修偏好标签、是否可注册、是否直租/直售、是否认证、状态、是否有上架房源、推荐标识及更新时间范围。城市改变后清空并重载下级区域、商圈、地铁；面积和价格筛选针对关联上架房源的聚合范围，最小值不得大于最大值。默认仅显示状态为启用，按最近更新时间倒序；重置恢复默认，合法筛选值及排序写入 URL。

## 7. 列表或内容展示
每行展示封面缩略图、楼盘名称、楼盘自有的可注册/楼盘认证标签、房源派生的直租/直售标签、城市/区/商圈/最近地铁、地址、楼盘类型、在租/在售面积段、可租/可售/直租/直售上架套数、租赁单价与总月租范围、出售单价与总价范围、顾问、推荐序号、启用状态、更新时间和操作。面积以平方米显示一位小数；租赁按币种、计价单位和计价周期分组展示，出售按币种分别显示单价和总价，租售及不同口径不得混算。空值显示“-”，金额和顾问按权限展示。默认按推荐序号升序、更新时间降序，可按名称、各类套数、同口径价格和更新时间排序。

## 8. 详情及表单
基础信息必填：名称、城市、区、商圈、详细地址、楼盘类型/等级、状态；别名和竣工年月可选。名称在同城市下按标准化名称（去首尾空格、全半角统一、忽略大小写）唯一。供给关系中 `default_merchant_id` 必填，Building 所属城市和区域必须启用，商户必须业务状态启用、资质已通过且未过期、服务城市已启用并覆盖本楼盘城市；默认商户关系使用 `[effective_from,effective_to)`，`effective_to` 可空且非空时 `effective_from < effective_to`，保存商户 ID/名称/资质/服务城市快照。数据库以 `building_id` 的区间排斥约束禁止关系重叠，空结束视为无穷，任一时点最多一条有效关系；变更只关闭旧关系并新建关系，不覆盖历史，`t=effective_to` 归属从该时刻开始的新关系。位置组保存地铁站及距离、经纬度、地图展示名；地图选点或合法经纬度至少其一。参数组包含总楼层、标准层面积、电梯和物业公司；数值字段必须非负并保存明确单位。Building 自有字段 `registration_capability` 必填且仅为固定机器枚举 `REGISTRABLE`（可注册）/`NOT_REGISTRABLE`（不可注册）/`PENDING_CONFIRMATION`（待确认），`building_verification_status` 必填且仅为 `UNVERIFIED`（未认证）/`VERIFIED`（已认证）/`EXPIRED`（认证失效）；认证必须关联证据编号和有效期；两者不得从房源反写或由字典标签决定。媒体最多 20 张、单张不超过 10MB，分类排序后首张为封面。配套含停车、会议室、前台、餐饮、健身、咖啡等及周边 POI；介绍、展示型特色标签、推荐楼盘关系和招商顾问可维护。顾问必须为启用经纪人，至少保留一名主顾问。

## 9. 核心操作与流程
新增保存时执行重复检测：命中同城同名或同坐标 100 米内的高相似记录，展示候选详情，操作者可取消、进入已有楼盘或在填写差异说明后继续创建；平台管理员可合并重复楼盘，目标保留不可变 ID，来源逻辑停用并迁移关联房源和审计链。编辑使用版本号提交；预览读取当前已保存内容及合格上架房源，不写入前台数据。停用前统计受影响房源，确认后撤销前台可见性，但不改写房源的 `publication_status` 或 `review_status`。批量启停、设置推荐或调整排序须显示影响数量、二次确认和逐条结果。导入先预校验并逐行返回错误；导出继承当前筛选、数据权限、字段权限和脱敏规则。

## 10. 状态与业务规则
楼盘状态为启用、停用、逻辑删除。Building 拥有名称、位置、地址、类型、图集、配套、介绍、顾问、推荐序号、`default_merchant_id` 及其生效关系、`registration_capability` 和 `building_verification_status`；Listing 拥有租售类型、价格、面积、楼层、单元标识、供给商户、`supply_visibility_hold`、直租/直售声明及可用性。一楼盘可关联多个房源且可经不同房源关联多个商户；Building 默认商户与 Listing 商户关系均使用 `[effective_from,effective_to)`，并分别由 `building_id`、`listing_id` 的数据库区间排斥约束保证任一时点唯一有效关系。Building 不持久化直租/直售标签、房源套数、面积段或价格范围，这些值只能由 Listing 聚合。房源 `publication_status` 仅为草稿/已上架/已下架/已出租，`review_status` 仅为未提交/待审核/审核通过/已驳回；楼盘、所属城市/区域、商户服务城市停用或 Listing 自身当前供给商户停用，或 `supply_visibility_hold=待复核`，只增加该房源的前台可见性约束，不改写两个字段。存在房源、线索或举报关联的楼盘不得物理删除。

## 11. 异常与边界
加载中显示骨架屏；无匹配数据展示空状态和新增入口；无权限按钮不展示且接口返回禁止。城市/商圈停用或删除时，既有楼盘保留原关联名称并标记失效，编辑必须迁移至启用节点。坐标缺失的楼盘可保存但地图预览提示不可展示。提交旧版本时拒绝覆盖，展示最新更新时间和刷新入口；上传、导入、导出失败提供失败原因及重试，不重复创建。合并遇到不同城市或目标已逻辑删除时禁止执行。

## 12. 数据与指标
主数据来自 Building、商户有效性及关系历史、位置字典、经纪人和 Listing；不可变键为 `building_id`。合格聚合房源必须复用 README 的有效供给谓词：未逻辑删除、`publication_status=已上架`、`review_status=审核通过`、`supply_visibility_hold=正常`、未被举报暂停且媒体完整；所属 Building、其城市和区域均启用；当前 Listing 商户关系有效，商户业务状态启用、资质已通过且未过期，商户服务城市已启用并覆盖楼盘城市。不满足任一条件即从前台、楼盘聚合、顾问与咨询路由候选中排除。`可租数量/可售数量` 分别对合格租赁/出售 `listing_id` 去重计数；`直租数量/直售数量` 再要求对应 Listing 的固定字段 `is_direct_listing=true`，数量大于 0 时才展示直租/直售声明。租赁/出售面积范围分别取对应 `area_sqm` 的最小/最大值。租赁单价最小/最大值仅在相同 `currency+rental_billing_unit+rental_billing_period` 分组内计算，总月租最小/最大值取 `rental_monthly_total`；出售单价和总价最小/最大值分别取 `sale_unit_price`、`sale_total_price` 并按 `currency` 分组，禁止跨币种换算。可注册标签仅在 `registration_capability=REGISTRABLE` 时展示；楼盘认证标签仅在 `building_verification_status=VERIFIED` 且证据未过期时展示。聚合缓存最长 5 分钟，商户启停、资质/服务城市变化、城市/区域启停和复核冻结变更须立即失效。

## 13. 埋点、通知与审计
埋点：`building_search`、`building_create`、`building_edit`、`building_preview`、`building_enable`、`building_disable`、`building_merge`、`building_export`，记录操作者、城市、结果和耗时。停用有上架房源时向楼盘主顾问和房源创建人发送站内提醒；默认商户变更、商户停用、资质失效或复核冻结时记录影响 Listing 数并通知关联房源创建人；合并向来源/目标关联房源的创建人提醒。审计记录对象 ID、动作、前后值、默认商户关系生效期、边界归属及不可变快照、原因、影响房源数、操作者、IP、时间和请求 ID，媒体排序和导出同样可追溯。

## 14. 验收标准
正常：运营人员只能以同城有效商户创建含排序图集、坐标、顾问、配套、注册能力和认证证据的启用楼盘，合格房源变化后 5 分钟内刷新套数、面积及价格范围。权限：运营人员不能查询或导出未授权城市，未获启停或导出操作权限者调用对应接口返回 403；销售主管、经纪人和客服不能越过各自只读或无入口边界。异常：无有效默认商户、默认商户资质失效、服务城市不启用/不覆盖、楼盘城市或区域停用时拒绝保存；同城重复名称保存前出现候选，旧版本编辑被拒绝，停用含已上架房源经确认后前台不可见且不改写房源状态。数据：构造相邻 `[from,to)` 默认商户关系验证 `t=to` 由新关系确定，构造重叠关系验证数据库排斥约束拒绝；用构造数据验证 Building 自有标签不被 Listing 反写，且 `supply_visibility_hold=待复核`、城市/区域停用或商户服务城市停用均从聚合排除；同楼盘可存在不同商户的多个 Listing，直租/直售只按合格 Listing 的固定声明派生，各租售计数按 `listing_id` 去重，面积和四类价格 min/max 与第 12 节谓词、分组完全一致。

## 15. 依赖与风险
依赖城市/商圈/地铁字典、地图地理编码、媒体存储、经纪人管理、房源状态聚合、前台预览和操作日志。地图服务不可用时仍允许手工坐标保存；历史地址标准化不足会提高重复误判风险，候选只提示不自动合并。MVP 不提供广告推荐位或面向用户的楼盘内容管理系统，推荐仅为基础标识和排序。

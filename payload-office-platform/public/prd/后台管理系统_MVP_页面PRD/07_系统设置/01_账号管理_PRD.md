## 2. 页面概述

**业务背景：** 后台账号是业务权限、城市范围、团队归属和审计主体的唯一入口。账号管理统一维护所有内部登录主体，业务页面不得自行创建或维护账号、密码、角色和数据范围。

**页面目标：** 让平台管理员安全完成账号邀请、创建、组织绑定、角色授权、停用和会话处置，并使变更可追溯。

**成功标准：** 合法账号可按已授予角色和数据范围访问；重复身份、停用账号、锁定账号和无效会话均不能继续使用后台。

**MVP 范围：** 账号邀请/创建、编辑、角色与城市范围绑定、团队绑定、启停用、重置密码、会话失效、敏感字段访问授权、锁定状态查看与解锁。

**非目标：** 不建设员工入转调离同步、SSO、MFA、外部合作方自助注册或复杂审批流。

## 3. 用户与权限

| 项目 | 规则 |
|---|---|
| 适用角色 | 仅拥有“系统设置/账号管理”菜单权限的账号可进入；默认仅平台管理员拥有全部能力。 |
| 数据范围 | 查询、编辑和导出均受账号管理数据权限限制；具备账号管理全局权限的平台管理员可管理全部账号；具备相应受限权限的平台管理员或获授权角色仅可管理授权城市和团队内账号，且不得管理权限等级高于或等于自己的账号。 |
| 操作权限 | 分为查看、新增、编辑、启停用、重置密码、失效会话、解锁、查看敏感字段、导出八项独立操作权限。 |
| 敏感信息 | 手机号默认脱敏；仅配置了对应字段权限的人员可查看完整手机号。密码哈希、原始密码、会话令牌永不展示或导出。 |

账号权限由其全部启用角色并集计算；数据范围按角色规则合并后再受账号城市、团队绑定的限制，详见《02_角色权限_PRD》。

## 4. 页面入口与关联

| 项目 | 规则 |
|---|---|
| 菜单入口 | 系统设置 > 账号管理。 |
| 前置条件 | 操作人已登录、账号管理菜单和对应操作权限均有效。新增时至少存在一个可分配的启用角色。 |
| 上游来源 | 角色权限提供角色与字段/数据权限；城市区域提供城市范围；团队管理提供团队；商户与经纪人管理可提供业务身份关联。 |
| 下游去向 | 可打开账号详情抽屉、角色详情、团队详情、关联经纪人/商户资料和操作日志；所有跳转携带对象不可变 ID。 |
| 返回规则 | 从列表打开的详情或编辑关闭后，保留原筛选、页码、排序和滚动位置；保存成功刷新当前列表。 |

## 5. 页面结构

1. 顶部为标题、账号总数/启用数/锁定数统计和“邀请账号”“新建账号”操作区。
2. 筛选区默认展开，包含查询、筛选、重置；筛选条件写入 URL。
3. 中部为账号列表，默认按创建时间倒序，每页默认 20 条，可切换 20/50/100 条。
4. 行内操作为详情、编辑、重置密码、失效会话、启用/停用及更多菜单；无权限操作不展示。
5. 详情、创建和编辑使用右侧抽屉；危险操作使用确认弹窗。窄屏时筛选区折叠、表格横向滚动，不省略字段权限提示。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 关键词 | 姓名、登录账号或手机号模糊匹配；手机号按操作人字段权限决定是否可检索完整值。 |
| 角色 | 启用/停用角色均可作为历史筛选条件；默认不限。 |
| 城市范围 | 仅展示操作人可管理的启用城市；可筛选“未绑定城市”。 |
| 所属团队 | 来自团队管理；先选城市时只展示该城市服务范围内团队。 |
| 账号状态 | 待激活、启用、停用、锁定；默认不限。创建邀请成功后，待激活账号必须立即可按此条件、关键词和不可变账号 ID 检索。 |
| 敏感字段授权 | 有/无“查看完整手机号”等字段权限。 |
| 最近登录时间 | 北京时间日期范围，开始不得晚于结束；默认不限。 |

点击查询后从第 1 页加载；重置恢复默认条件并清除 URL 参数。筛选变化不改变当前账号状态，导出继承当前条件和数据权限。

## 7. 列表或内容展示

| 字段 | 展示及规则 |
|---|---|
| 姓名/身份 | 姓名、关联经纪人或商户身份；无关联显示“-”。 |
| 登录账号 | 唯一登录名，可复制；不允许以手机号替代内部 ID。 |
| 手机号 | 按字段权限显示完整值或 `138****1234`。 |
| 角色 | 展示启用角色标签；角色停用时保留“已停用”标记。 |
| 城市范围/团队 | 城市名称列表和所属团队；空值显示“未绑定”。 |
| 状态 | 待激活、启用、停用、锁定；待激活显示邀请发送时间、链接有效期和是否已过期，锁定同时显示剩余锁定时间。 |
| 最近登录 | 北京时间，精确到分钟；从未登录显示“从未登录”。 |
| 创建时间/更新人 | 北京时间，精确到分钟；创建人和最近修改人可查看详情。 |
| 操作 | 根据权限显示，不以状态文案替代实际权限校验。 |

列表支持姓名、登录账号、状态、最近登录和创建时间排序。每行主键为不可变账号 ID，名称、手机号或账号变更不影响关联业务数据。

## 8. 详情及表单

### 8.1 创建与邀请

| 分组 | 字段与校验 |
|---|---|
| 身份信息 | 姓名必填，1-50 字；手机号必填且中国大陆手机号格式、全局唯一；登录账号必填，4-32 位字母/数字/下划线、全局唯一。 |
| 账号方式 | 选择“邀请”时生成一次性激活链接；选择“后台创建”时生成临时密码。临时密码只在成功弹窗展示一次，必须首次登录修改。 |
| 组织绑定 | 商户、团队、关联经纪人可选；经纪人和销售主管身份必须绑定有效团队。关联对象必须启用且在可管理范围内。 |
| 角色与范围 | 至少选择一个启用角色；可绑定城市范围。城市为空仅在角色本身拥有全局范围时允许，否则阻止保存。 |
| 状态 | 新建由系统按账号方式写入默认状态：后台创建为启用，邀请未激活为待激活且不计为可登录账号；创建表单不提供状态编辑控件。 |

### 8.2 编辑与详情

通用编辑表单仅可编辑姓名、手机号、组织绑定、角色和城市范围，不得展示、提交或通过接口修改账号状态；启用、停用必须分别通过专用动作发起。登录账号创建后不可修改，需新建账号替代。详情显示不可变账号 ID、创建/修改信息、最近登录、锁定信息、有效会话数、敏感字段授权来源和关联对象。编辑采用版本号校验，旧版本提交时阻止覆盖并提示刷新。

## 9. 核心操作与流程

| 操作 | 流程与结果 |
|---|---|
| 邀请/创建 | 提交通过校验后创建账号、角色关系和审计记录；邀请方式创建后状态为待激活，列表和状态筛选立即可检索。邀请链接 24 小时有效且仅可使用一次，失效后可重发；受邀人通过有效链接设置初始密码后，以原子操作完成待激活→启用、消耗链接、写入激活时间和审计，任一步失败则保持待激活且链接不消耗。后台创建仍直接为启用并按首次登录改密规则处理。 |
| 编辑角色或范围 | 展示权限预览和受影响业务范围；保存后立即使服务端权限生效，并强制该账号所有会话失效，提示其重新登录。 |
| 重置密码 | 二次确认后生成一次性重置链接或临时密码；立即失效旧会话并记录操作者、对象和结果。不得由操作人设置或查看原密码。 |
| 失效会话 | 二次确认后撤销目标账号全部有效会话，不改变账号状态；目标账号下次访问要求重新登录。 |
| 锁定/解锁 | 连续密码错误 5 次锁定 15 分钟；具备解锁权限的平台管理员或获授权角色可提前解锁。解锁不重置失败审计记录。 |
| 停用 | 显示该账号负责的客户、待办和团队职责数量；经纪人/销售主管存在未完成客户或唯一团队主管职责时，必须先完成转派并确认影响。确认后，以单一原子操作完成前置业务转派/影响确认、账号状态变更为停用、撤销全部有效会话及刷新令牌、从可分配候选人中移除和审计写入；任一步失败即整体失败并回滚，不得形成半停用，历史归属和数据保留。 |
| 启用 | 校验至少一个启用角色、必要团队和城市范围仍有效；通过后允许新登录，不恢复已撤销会话。 |

所有创建、编辑、角色变更、密码重置、解锁、会话失效和状态变更写入操作日志；停用日志须记录原子操作结果、转派/影响确认、状态变更及会话和刷新令牌撤销结果。高风险操作必须输入目标登录账号二次确认。

## 10. 状态与业务规则

| 状态 | 定义与流转 |
|---|---|
| 待激活 | 已邀请但未设置初始密码；不可登录、不可被分配新业务，可重发邀请或停用。仅可由有效且未使用的邀请链接完成初始密码设置后原子流转至启用；链接过期、无效或激活写入失败时保持待激活，重发只作废旧链接而不改变状态。 |
| 启用 | 可在角色、城市范围和团队规则内登录及操作。 |
| 锁定 | 密码连续错误达到阈值后的临时状态；到期自动恢复启用，或由具备解锁权限的平台管理员或获授权角色解锁。 |
| 停用 | 不可登录、不可被分配新业务；可重新启用，历史数据和日志保留。 |

- 手机号、登录账号均在全量账号（含停用）中唯一；停用后不可复用。
- 城市、团队、角色或关联身份停用后，账号保留历史显示但不得依赖该失效配置获得新权限；系统提示具备账号管理权限的平台管理员或获授权角色修正绑定。
- 账号不得修改、删除或停用自身；最后一个启用且具备全局系统设置权限的平台管理员不得停用或降权。
- 账号的敏感字段访问由角色字段权限决定，不能在本页面绕过角色授权单独加白。
- 状态流转仅为：邀请创建→待激活→启用；后台创建→启用；启用↔停用；启用→锁定→启用。待激活可直接停用，停用账号重新启用后进入启用而非待激活；通用编辑接口不得改变上述任一状态。

## 11. 异常与边界

| 场景 | 处理 |
|---|---|
| 加载/查询失败 | 保留筛选和列表旧数据，展示失败提示与重试按钮。 |
| 无权限 | 菜单不可见；通过 URL 访问返回无权限页，不泄露账号信息。 |
| 重复身份 | 服务端再次校验登录账号和手机号唯一性，定位字段并给出冲突提示。 |
| 角色或团队已失效 | 提交阻止，提示刷新配置后重选；详情保留历史标识。 |
| 并发编辑 | 版本号不一致时拒绝写入，显示最近更新人和时间。 |
| 邀请或重置链接失效 | 显示失效原因，可由具备权限者重新发起；待激活账号保持可检索，重发邀请仅作废旧链接并保持待激活；旧链接不可再次使用。 |
| 停用影响未处理业务 | 展示影响数量和转派入口；未完成转派不允许停用。 |
| 停用原子操作失败 | 前置转派/影响确认、状态变更、会话或刷新令牌撤销任一环节失败时，返回失败并整体回滚；账号保持原状态，旧会话与刷新令牌保持原有效状态，不得留下部分转派、已停用或部分撤销结果。 |
| 通过编辑提交状态 | 通用编辑页面不展示状态控件，服务端拒绝编辑接口携带的状态字段；必须改走启用或停用专用动作。 |

## 12. 数据与指标

账号数据来源为身份服务、角色权限、城市区域、团队、商户/经纪人关系和会话服务。顶部统计以当前操作人数据范围内的账号为口径：账号总数为未逻辑删除账号，启用数为状态“启用”，锁定数为查询时仍在锁定期的账号。列表和统计实时查询；最近登录写入后 5 分钟内可见。导出仅支持具备账号导出权限的人员，采用当前筛选和字段权限，手机号按权限脱敏，导出行为必须写日志。

## 13. 埋点、通知与审计

| 类型 | 内容 |
|---|---|
| 埋点 | `account_list_view`、`account_create_submit`、`account_invite_sent`、`account_role_changed`、`account_disabled`、`account_password_reset`、`account_session_invalidated`。 |
| 通知 | 邀请、密码重置、停用、角色/数据范围变化和会话失效通过站内消息通知目标账号；MVP 不发送短信或微信。 |
| 审计 | 记录操作者、操作者角色/组织、目标账号 ID、操作、变更前/后（密码与令牌不记录原文）、请求 ID、IP、设备、结果、失败原因和北京时间。 |

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 具备账号管理权限的平台管理员或获授权角色可邀请账号并完成激活；邀请创建后账号状态为待激活，可立即按状态、关键词和账号 ID 检索，受邀人完成有效链接的初始密码设置后原子变为启用；创建后可绑定多个角色、城市和团队；重置密码后旧会话失效。 |
| 权限 | 无账号管理菜单的用户不能访问；具备相应受限权限的平台管理员或获授权角色不能管理超出城市/团队范围或更高权限账号；无字段权限时手机号脱敏；通用编辑表单及编辑接口均不能修改状态。 |
| 异常 | 重复手机号或账号不能提交；角色失效、版本冲突、邀请过期和接口失败均有明确提示且不产生错误写入；停用任一原子环节失败时不产生半停用。 |
| 数据 | 停用成功后账号立即不能登录、不可被分配，全部旧会话和刷新令牌立即失效；前置转派/影响确认、状态变更与令牌撤销必须同时成功。修改角色后重新登录按新权限生效；所有敏感操作可在操作日志按账号 ID 和请求 ID 查到。 |

## 15. 依赖与风险

依赖身份认证、密码哈希与会话撤销服务、角色权限、城市区域、团队、商户/经纪人、消息和操作日志。风险包括停用或降权影响在办业务、城市范围与角色范围冲突、敏感信息导出泄露；上线前须配置至少两个启用且具备全局系统设置权限的平台管理员、密码策略、邀请域名和会话撤销接口。

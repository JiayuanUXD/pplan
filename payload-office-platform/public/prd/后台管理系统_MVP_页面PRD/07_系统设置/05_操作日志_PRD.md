## 2. 页面概述

**业务背景：** 楼盘、房源、客户线索、商户、经纪人、团队、账号、角色、区域和字典的关键变更及敏感读取必须统一审计。业务页面可写审计事件，但不可自行删除、改写或定义独立日志规则。

**页面目标：** 以不可篡改的日志记录操作人、对象、前后值、请求上下文和执行结果，支持授权查询、问题追溯、导出与留存。

**成功标准：** 可按业务对象和请求定位一次操作；敏感字段在列表、详情和导出中按权限掩码；普通用户无法编辑或删除日志。

**MVP 范围：** 操作人、角色、组织/团队、动作、对象类型/ID、前后值脱敏、IP/设备、请求 ID、结果/失败原因、北京时间、筛选、详情、导出、留存、不可变性、访问权限和有效业务对象链接。

**非目标：** 不建设 SIEM 对接、实时告警编排、全量页面浏览行为回放、数据库审计、日志修复或自助清理。

## 3. 用户与权限

| 项目 | 规则 |
|---|---|
| 适用角色 | 具备“系统设置/操作日志”菜单权限的人员可访问；默认仅具备日志审计权限的平台管理员可查看，也可通过自定义角色获得相应权限；日志审计职责不是新增默认角色。 |
| 数据范围 | 日志查询按操作日志数据权限和对象数据权限双重过滤。具备全局日志审计权限的平台管理员，或通过自定义角色获得相应权限的人员可查看所有日志；城市/团队受限的获授权角色仅查看其可见对象的日志及自身操作日志。 |
| 操作权限 | 查看列表、查看详情、查看敏感前后值、导出、查看失败原因和跳转业务对象分别控制；无新增、编辑、删除权限。 |
| 敏感信息 | 手机号、联系人、密码、会话令牌、身份证明、密钥等按字段掩码或完全排除。密码哈希和令牌永不记录为可读前后值。 |

## 4. 页面入口与关联

| 项目 | 规则 |
|---|---|
| 菜单入口 | 系统设置 > 操作日志；业务详情页的“查看操作日志”链接携带对象类型和对象 ID 进入。 |
| 前置条件 | 操作人已登录并具备日志菜单权限；日志服务、对象类型注册表和审计事件写入链路可用。 |
| 上游来源 | 全部业务模块、账号管理、角色权限、城市区域、字典配置、认证/会话服务和导出服务产生的审计事件。 |
| 下游去向 | 可跳转仍有效且当前用户有权限的楼盘、房源、线索、客户、商户、经纪人、团队、账号、角色、区域、字典等业务对象。 |
| 返回规则 | 从对象详情进入时保留对象过滤；跳转对象后返回日志页保留时间范围、页码、排序和详情抽屉状态。 |

## 5. 页面结构

顶部为日志总量、成功率、失败数和敏感操作数统计；其下是默认展开的筛选区。中部为分页日志表格，默认按操作时间倒序，每页 20 条，可切换 20/50/100。点击一行打开右侧详情抽屉，展示事件上下文和字段差异；有效对象显示跳转链接，失效或无权限对象显示只读标识。导出入口位于列表工具栏，仅具备导出权限时显示。窄屏将筛选收起、表格横向滚动，详情全屏展示。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 时间范围 | 北京时间，必选；默认近 7 天，单次最长 31 天，开始不得晚于结束。 |
| 操作人 | 姓名、登录账号或账号 ID；仅返回查看者有权知晓的操作人。 |
| 操作人角色 | 操作发生时的角色快照，支持系统/自定义角色筛选。 |
| 组织/团队 | 操作发生时的组织、团队快照；可筛选“未绑定”。 |
| 操作类型 | 新增、编辑、启停用、审核、驳回、上下架、分配/转派、导入、导出、登录、登出、密码重置、权限变更、会话失效等。 |
| 对象类型/ID | 已登记业务对象类型和不可变对象 ID；支持精确 ID 查询。 |
| 请求 ID（请求ID） | 精确查询，格式不合法时禁止提交。 |
| 结果 | 成功、失败、部分成功；失败可按失败原因关键词筛选。 |
| 来源 | 后台页面、API、批量任务、认证服务；IP 支持精确或 CIDR 范围检索（需专门权限）。 |
| 敏感操作 | 是否涉及敏感字段、权限变更、导出、密码/会话、状态变更。 |

筛选条件、排序和页码写入 URL。清空恢复近 7 天和默认倒序；导出严格继承当前筛选、日志数据范围和字段掩码权限。

## 7. 列表或内容展示

| 字段 | 展示及规则 |
|---|---|
| 操作时间 | 北京时间，精确到秒，按事件服务写入时间排序；显示时含时区 `+08:00`。 |
| 操作人 | 姓名、登录账号和账号 ID；已停用或已删除账号显示历史快照与 ID。 |
| 角色/组织/团队 | 操作发生时快照，不随之后配置变更改写。 |
| 操作 | 动作名称与敏感操作标识。 |
| 对象 | 对象类型、展示名快照、不可变对象 ID；有效且有权限时可跳转。 |
| 变更摘要 | 最多展示 3 个字段的“旧值 -> 新值”摘要；敏感值掩码。 |
| 请求上下文 | 请求 ID、来源、IP（按权限完整/部分掩码）、设备摘要。 |
| 结果 | 成功、失败、部分成功；失败显示简短原因码。 |
| 操作 | 查看详情；无详情权限时不可进入。 |

支持按操作时间、操作人、对象类型和结果排序；不得按已掩码内容做含糊排序。日志 ID 为不可变全局唯一 ID。

## 8. 详情及表单

本页无新增或编辑表单，原因是日志只能由服务端审计写入且不可人工改写。详情抽屉包含以下只读分组：

| 分组 | 字段与规则 |
|---|---|
| 事件标识 | 日志 ID、请求 ID、事件类型、操作时间（北京时间）、写入时间、幂等键。 |
| 操作主体 | 操作人账号 ID/名称/账号、角色快照、组织/团队快照、代理操作标记。 |
| 业务对象 | 对象类型、对象 ID、展示名快照、对象链接状态、父对象 ID（如房源所属楼盘）。 |
| 操作结果 | 成功/失败/部分成功、HTTP/业务结果码、失败原因、影响数量和逐条失败摘要（批量操作）。 |
| 前后值 | 字段级变更前/后值、字段标签、数据类型；敏感字段按查看权限掩码，密码哈希/令牌/密钥显示“不可展示”。 |
| 请求环境 | 来源页面/接口、IP、设备类型、User Agent 摘要、客户端请求 ID、服务端请求 ID。 |

前后值按字段白名单记录：业务关键字段、状态、归属、权限、范围、标签和金额/面积等。仅变更字段写入差异；新增只记录后值，停用/逻辑删除记录状态变化，不复制无关大字段或附件二进制内容。

## 9. 核心操作与流程

| 操作 | 流程与结果 |
|---|---|
| 记录审计事件 | 业务服务在写入关键操作的同一请求中提交结构化事件，包含操作者、对象、差异、上下文和结果；失败操作也必须记录。 |
| 查询列表 | 先按日志访问范围过滤，再按对象权限二次过滤；结果按操作时间倒序返回。 |
| 查看详情 | 服务端再次校验详情和敏感前后值权限；读取不可改变访问时间以外的日志内容。 |
| 跳转业务对象 | 校验对象是否仍有效及查看者对象权限；不可访问时留在详情并说明“对象已失效或无权限”。 |
| 导出 | 选择当前筛选或指定时间范围，确认数据量和掩码规则后异步生成；成功/失败均记录导出事件。 |
| 留存归档 | 到期日志由平台任务归档为只读文件/冷存储，保留可检索索引；归档不允许前台人工删除或改写。 |

以下必须产生日志：新增/编辑、审核/驳回、上下架、分配/转派、状态变更、权限/角色/范围变更、账号启停用、密码重置、会话失效、字典/区域配置变更、敏感字段查看、导入/导出和登录失败锁定。账号停用须在同一请求中记录前置转派/影响确认、状态变更、全部会话及刷新令牌撤销的原子结果；任一步失败仅记录失败结果，不得记录为成功或部分停用。批量操作记录批次总事件及逐条结果。

## 10. 状态与业务规则

| 状态 | 规则 |
|---|---|
| 成功 | 请求完成且全部目标已持久化；记录最终对象版本。 |
| 失败 | 请求未完成目标操作；记录已知失败原因、对象 ID 和请求上下文，不伪造前后值。 |
| 部分成功 | 批量操作部分完成；记录成功/失败数量和逐条结果索引。 |

- 日志采用追加写入（append-only）：无页面/API 的修改、删除、撤销能力；应用账户只能新增，数据库层通过权限和校验和保护历史记录。
- 每条日志保存操作时的主体、角色、组织、团队、对象显示名和权限快照；后续账号、角色、对象变更不回写历史。
- 时间统一保存 UTC 并以北京时间显示和导出；列表精确到秒，导出保留秒与 `+08:00` 偏移。
- 默认在线检索留存 180 天；归档留存 2 年。到期清理由受控平台任务执行并另记留存处理日志，不具备留存处理权限的平台管理员或获授权角色不能提前删除。
- 访问、查看敏感前后值和导出日志本身也产生审计事件，防止审计数据被无痕读取。
- 对象链接仅在对象存在、未被逻辑删除且查看者具备当前对象权限时有效；历史日志展示不依赖链接可用。

## 11. 异常与边界

| 场景 | 处理 |
|---|---|
| 审计写入失败 | 对高风险写操作采用事务外盒/可靠队列保证补偿；若仍不能记录，阻止该高风险操作并返回明确失败。低风险读取事件可异步重试并告警。 |
| 日志查询超时 | 保留筛选条件和旧结果，提示缩短时间范围或重试；复杂查询目标响应不超过 5 秒。 |
| 无权限 | 菜单不可见；URL 访问返回无权限；无敏感字段权限时详情和导出均掩码。 |
| 对象已失效 | 展示对象类型、ID 和历史快照，不提供跳转，不隐藏日志。 |
| 请求 ID 不存在 | 返回空状态及查询条件，不推断其他日志。 |
| 前后值不可解析 | 显示字段编码、原始数据类型和“无法渲染”，保留日志原始结构，不映射为错误值。 |
| 导出过大/失败 | 超出权限或 100,000 行上限时要求缩短范围；异步任务失败提供失败原因和重试，不泄露未授权行。 |

## 12. 数据与指标

数据来源为统一审计事件表、对象类型注册表、账号/角色/组织快照和对象权限服务。日志总量按当前可见事件计数；成功率 = 成功事件数 /（成功 + 失败 + 部分成功）事件数，部分成功单独展示；失败数和敏感操作数均以当前筛选和数据权限为口径。列表和详情实时查询热存储，归档日志可检索但可能异步加载。导出继承筛选、对象权限、字段掩码和 100,000 行上限，采用异步任务，下载链接有效 24 小时。

## 13. 埋点、通知与审计

| 类型 | 内容 |
|---|---|
| 埋点 | `audit_log_list_view`、`audit_log_detail_view`、`audit_log_sensitive_value_view`、`audit_log_object_link_open`、`audit_log_export_submit`、`audit_log_export_download`。 |
| 通知 | 审计写入连续失败、异常大量导出、权限变更或高风险敏感字段读取时，向具备日志审计权限的平台管理员或通过自定义角色获得相应权限的人员发送站内告警；日志审计职责不是新增默认角色，MVP 不接入外部 SIEM。 |
| 审计 | 本页的查询、详情访问、敏感值访问、导出、归档和留存处理均追加写入自身日志；严禁形成可修改的递归记录。 |

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 可按操作人、对象类型/ID、时间、结果和请求 ID 查询；详情正确显示前后值、角色/组织快照、IP/设备和有效对象链接。 |
| 权限 | 不具备日志审计权限的人员不能进入、编辑或删除日志；城市/团队受限的获授权角色只能看可见对象日志；无敏感权限时手机号、IP和前后值均按规则掩码。 |
| 异常 | 高风险操作在审计写入失败时被阻止；失效对象仍可查看历史日志但不能跳转；超大导出、查询超时和解析失败有明确恢复方式。 |
| 数据 | 审核、上下架、分配、导出、权限变更、账号会话处置均有完整日志；账号停用日志可同时核验前置转派/影响确认、状态变更与全部会话及刷新令牌撤销，任一步失败不出现成功或部分停用；日志不可变；时间以北京时间秒级显示；归档与留存按 180 天/2 年规则执行。 |

## 15. 依赖与风险

依赖所有业务写服务、认证/会话服务、统一审计事件 SDK、对象权限服务、异步导出、存储归档、消息通知和时钟服务。风险包括审计失败造成合规缺口、日志量增长导致查询慢、前后值泄露敏感信息、对象权限变化导致追溯困难；上线前须完成高风险事件清单、字段掩码白名单、可靠写入演练、180 天热存与 2 年归档配置。

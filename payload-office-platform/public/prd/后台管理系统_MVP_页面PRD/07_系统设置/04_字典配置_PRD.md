## 2. 页面概述

**业务背景：** 楼盘、房源、客户线索、跟进、举报和审核所用枚举必须由一个字典中心统一管理。业务页面引用字典编码并展示当前或历史标签，禁止以展示文案硬编码业务状态或筛选项。

**页面目标：** 管理字典分类和字典项的展示标签、排序、前台可见性及停用保护，同时保证不可变编码和历史业务数据稳定。

**成功标准：** 启用字典可在规定业务表单和前台筛选中使用；停用后不再允许新增引用；改名、排序、可见性和状态均不破坏存量数据。

**MVP 范围：** 指定字典分类、字典编码、可编辑标签、排序、前台可见性、停用安全检查、使用数量和审计。

**非目标：** 不支持用户自定义任意分类、层级字典、国际化多语言、按城市覆盖值或批量导入导出模板。

## 3. 用户与权限

| 项目 | 规则 |
|---|---|
| 适用角色 | 具备“系统设置/字典配置”菜单权限可查看；仅全局管理员可新增、编辑、启停用和调整前台可见性。 |
| 数据范围 | 字典为全局共享配置，不按城市或团队隔离；业务列表引用字典时仍受自身数据权限控制。 |
| 操作权限 | 查看、新增、编辑、排序、启停用、查看使用数量、导出和查看日志独立控制。 |
| 敏感信息 | 字典值不含个人敏感数据；使用明细中的客户、账号等对象字段按对象字段权限脱敏。 |

## 4. 页面入口与关联

| 项目 | 规则 |
|---|---|
| 菜单入口 | 系统设置 > 字典配置。 |
| 前置条件 | 操作人具备菜单及操作权限；系统已登记本页允许维护的字典分类。 |
| 上游来源 | 无业务数据上游；字典分类和编码规范由系统发布并受版本控制。 |
| 下游去向 | 楼盘、房源、线索、跟进、举报、审核、前台筛选、统计筛选和操作日志展示。 |
| 返回规则 | 从使用明细或日志返回时保留分类、状态、关键词、页码和排序。 |

## 5. 页面结构

左侧为字典分类导航及每类启用/停用项数量；右侧依次为筛选区、字典项表格、分类说明和编辑抽屉。默认选中“楼盘类型”，列表按排序升序展示。顶部仅提供新增字典项，分类由系统登记，不在 MVP 中由页面创建。行内提供编辑、排序、启停用、使用明细和日志。窄屏将分类导航折叠为选择器，列表横向滚动并保留状态和使用数量列。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 字典分类 | 固定分类清单，切换后刷新当前分类项；必选。 |
| 关键词 | 标签、字典编码和备注模糊匹配。 |
| 状态 | 启用、停用；默认展示启用。 |
| 前台可见性 | 可见、不可见、全部；只对支持前台展示的分类生效。 |
| 是否已使用 | 按全量有效业务引用计算，支持“未使用”“已使用”。 |
| 系统内置 | 是/否；内置项可限制编辑范围。 |

筛选条件写入 URL；重置恢复当前分类的默认筛选。导出继承筛选和权限，不能绕过字典分类限定。

## 7. 列表或内容展示

| 字段 | 展示及规则 |
|---|---|
| 字典分类 | 展示中文分类和机器分类键；分类键不可编辑。 |
| 标签 | 面向后台和前台的中文展示名，可编辑；历史记录仍显示其保存时快照和编码。 |
| 字典编码 | 全局唯一的机器稳定值，创建后不可修改，格式为大写字母、数字、下划线，2-64 位。 |
| 排序 | 非负整数，数值越小越靠前；同值稳定排序。 |
| 前台可见 | 支持前台筛选/展示的分类可配置；状态停用时强制不可见。 |
| 状态 | 启用/停用；停用不改变历史业务数据的编码。 |
| 使用数量 | 当前有效业务对象对该编码的引用数，可按对象类型查看明细。 |
| 系统内置/更新时间 | 内置标识、修改人、北京时间精确到分钟。 |

## 8. 详情及表单

### 8.1 MVP 字典分类及归属

| 分类 | 使用页面/字段 |
|---|---|

| 装修状态 | 房源列表/编辑及前台筛选：毛坯、简装、精装、带家具等。 |
| 房源标签 | 房源和楼盘展示：近地铁、可注册、直租、实勘、免佣、认证等。 |
| 注册属性 | 楼盘/房源：可注册、不可注册、待确认。 |
| 直租/认证标签 | 楼盘/房源前台标识：直租、直售、已认证、品牌直租。 |
| 线索来源 | 线索池：前台表单、电话咨询、委托找房、导入、客服等。 |
| 线索阶段 | 线索页：新建、待分配、跟进中、有效商机、带看、谈判、已转化、已流失；属于受保护状态机，MVP 不允许在本页新增、删除、停用或改变流转顺序。 |
| 流失原因 | 线索阶段转为已流失时必填。 |
| 跟进方式 | 跟进记录：电话、微信、面谈、带看、系统记录等。 |
| 跟进结果 | 跟进记录：已接通、未接通、已确认需求、已推荐房源、已预约带看、待再次跟进等；结果标签不得直接改写线索阶段。 |
| 举报类型 | 房源举报：虚假信息、重复、价格异常、已出租等。 |
| 举报处理结果 | 举报处理：有效、无效、已下架、需补充资料等。 |
| 驳回原因 | 房源审核和举报处理的驳回/退回原因。 |

### 8.2 字典项表单

| 字段 | 规则 |
|---|---|
| 字典分类 | 新建时必选固定分类，保存后不可变。 |
| 标签 | 必填 1-50 字；同一分类下标签唯一。 |
| 字典编码 | 必填、全局唯一、创建后不可变；系统内置编码不可修改。 |
| 排序 | 可选非负整数，默认追加至分类末尾。 |
| 前台可见 | 仅前台适用分类可选，默认否；启用状态下才可为是。 |
| 状态 | 新建默认启用；系统内置项由发布策略决定是否允许停用。 |
| 备注 | 可选 0-500 字，说明适用规则，不作为前台文案。 |

## 9. 核心操作与流程

| 操作 | 流程与结果 |
|---|---|
| 新增字典项 | 选择系统允许分类，填写标签、编码、排序等；通过唯一性校验后创建，默认前台不可见。 |
| 编辑标签/排序/可见性 | 展示使用数量和前台影响，保存后后台候选项实时刷新、前台缓存 5 分钟内刷新。 |
| 查看使用数量 | 按楼盘、房源、线索、跟进、举报、审核等对象分组展示引用数；可跳转有权限的业务对象。 |
| 停用 | 展示使用数量和影响；确认后禁止在新增/编辑表单选择、从前台筛选移除，历史记录继续展示标签快照和编码。 |
| 启用 | 校验分类仍启用且编码有效；启用后可用于业务新建，前台是否出现由可见性决定。 |
| 排序 | 支持拖拽或输入排序值，影响同类后台下拉和前台筛选顺序，不重写业务对象编码。 |

不提供物理删除。系统内置且被核心状态机依赖的项（如线索阶段）不得停用；需要下线时必须先完成业务迁移和版本发布。修改标签、启停用、前台可见性和排序均写入操作日志。

## 10. 状态与业务规则

| 状态 | 规则 |
|---|---|
| 启用 | 可在所属业务的新增/编辑表单中选择；是否在前台展示由前台可见性控制。 |
| 停用 | 不得在新增/编辑中选择，前台不可见，历史数据保留原字典编码及标签快照。 |

- 业务对象必须持久化字典编码，不得持久化标签作为唯一值；展示时优先显示历史快照，缺少快照时按编码解析当前或“已停用”标签。
- 字典编码创建后不可编辑、不可复用；标签可编辑但不改变编码含义。
- 每个分类下标签唯一，全局编码唯一；排序只在同分类内生效。
- 使用数量大于 0 的项不可删除；MVP 全部字典项均不提供删除，以停用替代。
- 前台可见性仅控制前台筛选/标签，不控制业务后台是否可用；但停用必然强制前台不可见。
- 线索阶段、驳回原因等状态机所需分类必须由业务流程校验，不能通过任意停用绕过必填原因规则。

## 11. 异常与边界

| 场景 | 处理 |
|---|---|
| 无权限 | 菜单不可见；直接访问返回无权限，不返回分类和值。 |
| 标签/编码冲突 | 服务端唯一校验，定位字段并给出冲突原因。 |
| 系统内置受限 | 不允许编辑编码或停用时，禁用操作并说明由系统发布管理。 |
| 已使用/状态机依赖 | 不允许删除；停用前展示引用和依赖，核心状态机项阻止停用。 |
| 并发编辑 | 版本号冲突时拒绝覆盖并要求刷新。 |
| 前台缓存发布失败 | 后台保存成功但提示前台同步失败并告警；重试发布，不回滚编码和审计记录。 |
| 字典解析缺失 | 历史列表显示保存快照或“未知编码（CODE）”，不得静默映射为其他标签。 |

## 12. 数据与指标

字典分类、字典项、业务对象字典编码、历史标签快照和引用统计为数据来源。使用数量为当前未逻辑删除业务对象对该编码的关联数，按对象类型聚合；历史日志不计入使用数量。后台字典候选项实时读取，前台可见项通过缓存发布，最长 5 分钟。具有字典导出权限的管理员可导出当前分类筛选结果、编码、标签、排序、状态、可见性和使用数量，导出操作写入日志。

## 13. 埋点、通知与审计

| 类型 | 内容 |
|---|---|
| 埋点 | `dictionary_list_view`、`dictionary_create_submit`、`dictionary_edit_submit`、`dictionary_usage_view`、`dictionary_disabled`、`dictionary_visibility_changed`。 |
| 通知 | 停用被使用字典、修改前台可见标签或状态机相关字典时，向业务配置管理员发送站内通知。 |
| 审计 | 记录操作者、角色/组织、分类、字典项 ID/编码、变更前后标签和状态、使用数量、请求 ID、IP、设备、结果、失败原因和北京时间。 |

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 可在允许分类新建编码不可变的字典项；标签、排序和前台可见性修改正确同步到后台候选和前台缓存。 |
| 权限 | 无字典维护权限者不可写入；无业务对象权限者只能看到使用数量，不能打开敏感引用明细。 |
| 异常 | 重复编码/标签、非法排序、系统内置受限和状态机依赖项停用均被阻止并提示原因；并发编辑不覆盖。 |
| 数据 | 停用后新增/编辑表单不可选择而历史数据仍显示原标签；使用数量按编码关联准确；每次配置变更均可追溯至操作日志。 |

## 15. 依赖与风险

依赖楼盘、房源、线索、跟进、举报、审核、前台筛选缓存和操作日志。风险包括误改标签造成前台文案变化、误停用阻塞业务录入、编码被错误复用导致统计混乱；上线前须登记分类/编码基线、核心状态机保护清单和前台缓存发布策略。

## 2. 页面概述

**业务背景：** 全部业务页面共享菜单、操作、字段和数据权限。角色权限是唯一授权源，业务 PRD 只能声明所需权限点，不得自行维护角色、权限枚举或数据范围。

**页面目标：** 管理系统内置角色和自定义角色，实现最小权限、可预览、可复制、可审计的 RBAC 授权。

**成功标准：** 登录后的菜单、操作入口、敏感字段和可访问业务数据均与角色配置一致；高风险变更能确认影响并追溯。

**MVP 范围：** 系统/自定义角色、菜单/操作/字段/数据权限、范围继承和冲突处理、权限预览、复制、启停用、绑定账号数和高风险确认。

**非目标：** 不支持运行时自定义接口权限、条件表达式策略、临时授权、审批流或跨租户角色市场。

## 3. 用户与权限

| 项目 | 规则 |
|---|---|
| 适用角色 | 仅持有“系统设置/角色权限”菜单权限且具有角色查看权限的人员。 |
| 数据范围 | 角色为全局配置；仅拥有角色管理全局权限的平台管理员可创建、编辑、复制、启停用。 |
| 操作权限 | 查看、创建、编辑、复制、启停用、查看影响账号、预览权限、导出八项独立控制。 |
| 敏感信息 | 仅展示账号数量和可见范围内账号名称；手机号按查看者字段权限脱敏。权限配置本身属于敏感系统信息。 |

## 4. 页面入口与关联

| 项目 | 规则 |
|---|---|
| 菜单入口 | 系统设置 > 角色权限。 |
| 前置条件 | 权限注册表已发布；至少保留一个启用的“平台管理员”默认角色。 |
| 上游来源 | 系统菜单与操作权限注册表、字段权限注册表、数据范围模型、账号管理、城市区域和团队管理。 |
| 下游去向 | 账号管理分配角色；所有业务页面按授权渲染；角色详情可跳转受影响账号和操作日志。 |
| 返回规则 | 从受影响账号列表返回时保留来源筛选；保存角色后刷新当前角色详情和影响数量。 |

## 5. 页面结构

左侧为角色列表，按系统角色和自定义角色分组，显示状态与绑定账号数；右侧依次显示基本信息、权限标签摘要、权限树、数据范围配置和影响预览。顶部提供新建、复制、搜索和状态筛选；底部固定保存区。菜单/操作权限树默认展开一级菜单，字段和数据权限按业务模块折叠。窄屏以角色列表页与详情页切换呈现，保存和取消固定在底部。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 关键词 | 角色名称、角色编码模糊匹配。 |
| 角色类型 | 系统内置、自定义；默认不限。 |
| 状态 | 启用、停用；默认展示启用。 |
| 所属模块 | 工作台、楼盘、房源、客户线索、商户经纪、数据看板、系统设置；过滤权限树和列表摘要。 |
| 高风险权限 | 导出敏感字段、角色管理、账号管理、数据全量、操作日志导出等；支持仅看已授予角色。 |

重置后恢复默认筛选，URL 保留查询、类型、状态和模块。角色权限不支持按业务数据范围筛选，避免将全局配置误判为局部数据。

## 7. 列表或内容展示

| 字段 | 规则 |
|---|---|
| 角色名称/编码 | 名称可编辑，编码创建后不可变且全局唯一。 |
| 类型 | 系统内置或自定义；系统内置不可删除和不可改编码。 |
| 权限摘要 | 菜单数、操作数、字段权限数、数据范围类型及高风险权限数。 |
| 数据范围 | 全部城市、指定城市、所属团队、本人及组合规则摘要。 |
| 绑定账号数 | 当前绑定该角色的未逻辑删除账号数，可进入受影响账号列表。 |
| 状态 | 启用/停用；停用角色不参与新会话授权。 |
| 最近修改 | 操作人和北京时间，精确到分钟。 |
| 操作 | 详情、编辑、复制、启停用、查看影响、查看日志；依权限显示。 |

列表默认按系统角色优先、排序值、创建时间倒序；支持名称、绑定账号数、状态和最近修改排序。

## 8. 详情及表单

| 分组 | 字段与校验 |
|---|---|
| 基本信息 | 角色名称必填 2-30 字且唯一；角色编码必填，2-50 位小写字母、数字、下划线，创建后不可修改；描述 0-200 字；排序为非负整数。 |
| 类型与状态 | 新建只能为自定义角色；系统角色只可按受控范围编辑说明或配置，不得删除；状态默认为启用。 |
| 菜单权限 | 按一级/二级菜单勾选。授予操作、字段或数据权限时，必须同时拥有其所属菜单查看权限。 |
| 操作权限 | 按菜单下的新增、编辑、审核、上下架、分配、导入、导出、删除等注册权限点选择。 |
| 字段权限 | 按业务对象配置完整手机号、联系人、导出敏感字段、前后值查看等。未授予时服务端返回脱敏或拒绝字段。 |
| 数据权限 | 每个业务域可选择“全部城市”“指定城市”“所属团队”“本人”“无数据访问”；指定城市从启用城市选择，团队范围仅在账号有团队绑定时生效。 |

权限预览可选择一个账号和目标模块，展示可见菜单、可执行操作、敏感字段效果及数据范围解释；预览是模拟结果，不修改账号配置。

## 9. 核心操作与流程

| 操作 | 流程与结果 |
|---|---|
| 新建 | 填写基本信息并选择权限；保存前校验权限依赖和数据范围；成功后生成角色 ID 和审计记录。 |
| 编辑 | 显示变更前后权限差异、受影响账号数和高风险标记；版本号冲突时拒绝覆盖。 |
| 复制 | 选择源角色，复制权限和范围到新自定义角色；必须输入新名称与新编码，默认停用，避免未审阅即生效。 |
| 权限预览 | 选择账号或默认角色上下文后生成只读预览；对多角色账号显示各角色贡献和最终并集。 |
| 启用 | 校验至少有菜单权限和合法数据范围；启用后在目标账号下次鉴权时生效。 |
| 停用 | 展示绑定账号数、唯一角色覆盖账号数及会话影响；确认后角色不再参与新授权，受影响账号保留其他启用角色权限。 |

角色权限编辑、启停用和系统角色高风险变更须二次确认。变更保存后目标账号全部会话失效，强制重新鉴权。不得以批量静默方式授予高风险权限。

## 10. 状态与业务规则

### 10.1 角色状态

| 状态 | 规则 |
|---|---|
| 启用 | 参与账号的有效权限计算。 |
| 停用 | 不参与新会话权限计算；历史角色关系、业务记录和日志保留。 |

### 10.2 权限继承与冲突

- 账号可绑定多个角色。菜单、操作和字段权限采用**允许并集**：任一启用角色允许即允许。
- 数据范围按业务域采用**允许集合并集**：多个角色的可访问城市/团队/本人集合合并；“全部城市”覆盖同一业务域其他范围。
- 显式“拒绝”不进入 MVP；需要限制时不得同时绑定会扩大权限的角色。权限预览必须提示该风险。
- 操作权限依赖菜单权限；字段完整展示、敏感导出依赖对应菜单和操作权限；缺少上级权限的下级授权保存时自动阻止。
- 账号页面的城市绑定是最终边界，不会扩展角色给出的范围；团队数据范围须同时满足角色规则、账号团队绑定和服务城市约束。
- “平台管理员”默认角色必须保留全局系统设置和操作日志查看权限；最后一个启用且具备全局系统设置权限的平台管理员不得停用或移除其必要权限。

## 11. 异常与边界

| 场景 | 处理 |
|---|---|
| 无权限/越权 | 无菜单或写权限时拒绝访问/保存；服务端再次验证操作者不能修改高于自身的权限集合。 |
| 失效依赖 | 所选城市、团队、菜单或权限点已停用/下线时阻止保存并要求刷新。 |
| 权限依赖缺失 | 保存时定位缺失的菜单或操作依赖，提供勾选建议但不自动扩大数据范围。 |
| 并发修改 | 版本不一致时展示最近修改信息，拒绝覆盖。 |
| 停用后账号无角色 | 预览受影响账号；确认时要求将账号改绑至其他启用角色或同时停用账号。 |
| 接口失败 | 保留未保存选择，提示重试；未收到成功响应不得更新本地权限摘要。 |

## 12. 数据与指标

角色、权限点、角色权限关系、数据范围规则和账号角色关系均以不可变 ID 存储。绑定账号数为当前绑定角色且未逻辑删除的账号数；高风险权限数按已登记高风险权限点计算。权限配置实时读取，账号会话在变更后立即失效，重新登录后生效。角色配置不支持普通导出；仅具备角色导出权限的平台管理员或通过自定义角色获得相应权限的人员可导出，文件不包含完整手机号，且记录导出日志。

## 13. 埋点、通知与审计

| 类型 | 内容 |
|---|---|
| 埋点 | `role_list_view`、`role_create_submit`、`role_copy_submit`、`role_permission_preview`、`role_permission_changed`、`role_disabled`。 |
| 通知 | 角色启停用或权限变化时，对受影响账号发送站内消息，说明生效后需重新登录；不暴露其他角色配置明细。 |
| 审计 | 记录操作者、角色/组织、角色 ID、权限差异、影响账号数、确认文本、请求 ID、IP、设备、结果、失败原因和北京时间；字段权限日志不得泄露敏感字段原值。 |

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 可创建自定义角色、选择四类权限、复制后修改新角色；预览能正确展示多角色账号的最终权限。 |
| 权限 | 不具备角色管理全局权限的平台管理员或获授权角色不能编辑角色；缺少菜单权限时不能访问对应业务模块；字段权限未授予时手机号和导出字段均脱敏。 |
| 异常 | 缺少上级菜单的操作权限不能保存；停用唯一角色导致账号无可用角色时被阻止；并发保存不覆盖他人变更。 |
| 数据 | 全部城市与指定城市范围按并集正确生效但不突破账号城市边界；高风险权限、复制、启停用和变更前后均可在日志查询。 |

## 15. 依赖与风险

依赖统一权限注册表、鉴权服务、账号管理、城市区域、团队和操作日志。风险包括多角色并集导致权限扩大、误停用影响大量账号、权限点下线造成历史配置失效；上线前应完成权限点清单、“平台管理员”默认角色基线、平台管理员保底策略和会话失效联调。

## 2. 页面概述

**业务背景：** 城市、行政区、商圈、地铁线路和站点是楼盘、房源、线索、商户服务范围、经纪人分配、数据看板和前台位置筛选的统一地理主数据。业务页面只引用节点不可变 ID，不得重复维护区域树。

**页面目标：** 维护可用、可排序、可定位、可前台展示的城市区域层级，保证历史业务记录可追溯且不会被误删。

**成功标准：** 城市及下级区域可按规则联动选择；停用节点阻止新增业务引用但保留历史展示；前台仅展示已启用且允许前台展示的节点。

**MVP 范围：** 城市、行政区、商圈、地铁线路/站点、层级关系、代码、经纬度、排序、前台可见性、链接数据保护、启停用。

**非目标：** 不建设地图供应商管理、地理围栏、实时交通、SEO 页面生成、批量导入和跨城市行政区划自动同步。

## 3. 用户与权限

| 项目 | 规则 |
|---|---|
| 适用角色 | 具备“系统设置/城市区域”菜单权限的人员可查看；默认仅平台管理员可维护。 |
| 数据范围 | 地理主数据为全局共享配置，维护权限仅授予全局管理员；受限城市运营仅可按角色授权查看城市，不可改变层级或状态。 |
| 操作权限 | 查看、新增、编辑、排序、启用/停用、查看引用数据、查看日志、导出分别控制。 |
| 敏感信息 | 坐标为业务公开地理数据，不按手机号规则脱敏；精确坐标导出需单独导出权限。 |

## 4. 页面入口与关联

| 项目 | 规则 |
|---|---|
| 菜单入口 | 系统设置 > 城市区域。 |
| 前置条件 | 操作人具备菜单和操作权限；新增下级节点时上级节点必须为启用。 |
| 上游来源 | 无业务主数据上游；可引用受控的行政区编码规范和地图坐标服务。 |
| 下游去向 | 楼盘/房源的城市、行政区、商圈、地铁；线索意向区域；商户/经纪人服务范围；账号城市范围；数据看板与前台筛选。 |
| 返回规则 | 从引用数据或日志返回时保留树节点、筛选、表格页码和展开状态。 |

## 5. 页面结构

页面左侧为城市树：城市 > 行政区 > 商圈，城市下独立分组展示地铁线路 > 地铁站；右侧为当前节点列表/详情。顶部提供关键词、类型、状态、前台可见性筛选和新增入口。选中城市后默认展开其一级子节点；列表显示当前节点的直接子级并支持拖拽或输入排序。详情、新增和编辑使用抽屉，停用、排序冲突和引用检查使用弹窗。窄屏优先展示树，选中节点后切至详情并保留面包屑返回。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 关键词 | 节点名称、拼音、英文名、不可变区域代码精确/模糊匹配。 |
| 城市 | 启用与停用城市均可用于历史查询；默认不限。 |
| 节点类型 | 城市、行政区、商圈、地铁线路、地铁站。 |
| 状态 | 启用、停用；默认展示启用。 |
| 前台可见性 | 可见、不可见；仅城市/行政区/商圈/地铁站可配置。 |
| 是否已引用 | 根据楼盘、房源、线索、账号、团队、商户、经纪人等有效关联计算。 |

筛选结果在树和列表同步高亮。点击重置恢复默认条件；筛选、选中节点和展开状态写入 URL。停用节点默认不出现在业务表单候选项，但可在本页筛选查看。

## 7. 列表或内容展示

| 字段 | 展示及规则 |
|---|---|
| 名称/类型 | 名称及城市、行政区、商圈、线路、站点类型；同一父节点下同类型名称不可重复。 |
| 层级/上级 | 面包屑展示；城市无上级，行政区和线路上级为城市，商圈上级为行政区，站点上级为线路且归属城市。 |
| 区域代码 | 系统内部唯一且创建后不可修改，用于业务关联和导入映射，不以名称作关联键。 |
| 坐标 | 经度、纬度保留 6 位小数；未配置显示“-”。 |
| 排序 | 非负整数，数值越小越靠前；同值按创建时间和 ID 稳定排序。 |
| 前台可见 | 是/否；仅启用节点可设置为是。 |
| 使用数量 | 分对象统计引用数量，可查看明细；统计按当前数据权限脱敏。 |
| 状态/更新时间 | 启用或停用；最近更新人和北京时间精确到分钟。 |

## 8. 详情及表单

| 字段 | 城市 | 行政区 | 商圈 | 地铁线路/站点 |
|---|---|---|---|---|
| 名称 | 必填，1-50 字 | 必填 | 必填 | 必填 |
| 上级 | 无 | 必填，启用城市 | 必填，启用行政区 | 线路上级为启用城市；站点上级为启用线路 |
| 区域代码 | 必填、全局唯一、创建后不可改 | 同左 | 同左 | 同左 |
| 拼音/英文名 | 可选，1-100 字 | 可选 | 可选 | 可选 |
| 坐标 | 可选，合法经纬度 | 可选 | 可选 | 站点建议必填，线路可选中心点 |
| 排序 | 可选，默认追加至末尾 | 同左 | 同左 | 同左 |
| 前台可见 | 可选，默认否 | 可选 | 可选 | 站点可选；线路不直接前台展示 |
| 简介/备注 | 可选，0-500 字 | 可选 | 可选 | 可选 |

经度范围为 -180 至 180，纬度范围为 -90 至 90；城市、行政区、商圈坐标为区域中心点，不替代楼盘精确坐标。编辑不得改变类型、区域代码和已建立的父子关系；层级调整属于非 MVP，需新建节点并迁移引用数据。

## 9. 核心操作与流程

| 操作 | 流程与结果 |
|---|---|
| 新增节点 | 选择类型与合法上级，校验名称/代码/坐标后保存；新节点默认启用但前台不可见，须显式确认后展示前台。 |
| 编辑 | 可编辑名称、别名、坐标、排序、可见性和备注；保存后业务表单和前台缓存于 5 分钟内刷新。 |
| 排序 | 拖拽或输入排序值；保存前预览同级顺序，冲突时按稳定排序处理。 |
| 查看引用 | 显示按对象类型聚合的使用数量，支持跳转楼盘、房源、线索等有效对象；无权限对象仅显示数量不显示详情。 |
| 停用 | 先检查自身、下级和业务引用。确认后停止出现在新增/编辑候选项、前台不可见；保留历史对象文本和 ID。 |
| 启用 | 校验所有上级均启用；站点还须线路启用。成功后可被业务新建引用，前台是否展示仍由可见性控制。 |

停用城市会同时阻止其行政区、商圈、线路和站点用于新建业务；不自动停用下级，以便历史维护和后续恢复。停用行政区时，其商圈及关联楼盘/线索不可新建引用；停用线路时站点不可新建引用。所有变更写入操作日志。

## 10. 状态与业务规则

| 状态 | 规则 |
|---|---|
| 启用 | 在父级启用的前提下可供新建/编辑业务表单选择；是否进入前台由“前台可见”独立控制。 |
| 停用 | 不可被新建或编辑业务选择，前台不可展示；历史楼盘、房源、线索、日志继续显示保存时的名称和 ID。 |

- 层级固定为“城市 > 行政区 > 商圈”和“城市 > 地铁线路 > 地铁站”；商圈与地铁站不可互为父级。
- 每个节点有全局不可变区域代码；名称、拼音、排序、坐标和可见性可在规则范围内编辑。
- 已被任何业务对象、账号范围、团队范围或下级节点引用的数据不允许物理删除；MVP 不提供删除操作。
- 前台位置筛选只读取启用且前台可见的节点，按排序升序；前台现有历史 URL 通过区域代码映射，不依赖名称。
- 城市、行政区、商圈和地铁站的使用数量必须从关联 ID 实时或准实时计算，不能通过名称匹配。

## 11. 异常与边界

| 场景 | 处理 |
|---|---|
| 无权限 | 无菜单时不可访问；无维护权限时只读且不返回超范围节点详情。 |
| 上级已停用 | 新增、启用或编辑关联时阻止提交，提示先启用上级。 |
| 名称/代码冲突 | 服务端唯一校验，定位冲突字段；同名异类型需明确选择类型和上级。 |
| 有下级或已引用 | 停用前展示分类型使用数量；禁止删除，允许在确认影响后停用。 |
| 并发排序/编辑 | 版本冲突时拒绝写入，刷新显示最新树和排序。 |
| 坐标无效/地图服务失败 | 坐标手工输入仍需范围校验；地图选点失败可重试，不得写入空或非法值。 |
| 前台缓存延迟 | 保存成功显示预计 5 分钟生效；若刷新失败记录告警且保留后台真实状态。 |

## 12. 数据与指标

数据来源为区域主数据表、父子关系、坐标字段和各业务对象的区域 ID 引用。使用数量口径为当前未逻辑删除的楼盘、房源、线索、商户、经纪人、团队和账号范围关联数，详情按对象自身数据权限过滤。区域查询实时读取；前台可见区域通过缓存发布，最长延迟 5 分钟。导出继承筛选和权限，包含代码、层级、状态和排序；精确坐标仅有坐标导出权限时包含，并记录审计日志。

## 13. 埋点、通知与审计

| 类型 | 内容 |
|---|---|
| 埋点 | `region_tree_view`、`region_create_submit`、`region_edit_submit`、`region_sort_saved`、`region_usage_view`、`region_disabled`、`region_front_visibility_changed`。 |
| 通知 | 城市或行政区停用、前台可见性变化时向相关城市运营及数据维护人员发送站内提醒；MVP 不发送外部通知。 |
| 审计 | 记录操作者、角色/组织、节点类型/ID/代码、父级、变更前后值、引用数量、请求 ID、IP、设备、结果、失败原因和北京时间。 |

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 可创建城市、行政区、商圈、线路和站点，层级联动正确；区域代码不可修改；排序改变后前台候选顺序一致。 |
| 权限 | 无维护权限者只读；受限人员不能修改全局树；无精确坐标导出权限时导出不含坐标。 |
| 异常 | 上级停用时不能新增下级；名称/代码重复和非法坐标不能保存；并发编辑不覆盖最新数据。 |
| 数据 | 停用城市/区域后楼盘、房源、线索不能新建引用但历史显示保留；前台只显示启用且可见节点；引用数量可追溯至关联 ID。 |

## 15. 依赖与风险

依赖楼盘、房源、线索、商户、经纪人、团队、账号、前台筛选缓存、地图坐标服务和操作日志。风险包括错误停用阻塞新业务、名称变更影响展示、区域调整导致历史归属混乱；上线前须导入首批城市树、定义区域代码规范、完成前台缓存失效和引用检查联调。

## 2. 页面概述

商户是与平台建立合作关系的企业、门店或品牌，是楼盘、房源和经纪人所属的合同主体；经纪人是商户下经身份与任职核验的个人，团队仅用于内部线索分配和数据范围，不能替代商户主体。

页面目标是维护合格商户及其服务范围，确保只有有效资质的商户可供给房源和承接咨询。成功标准为：每个新增或启用商户均有可追溯的资质核验记录，停用不会留下可展示的失效顾问、无路由线索或未经处置的关联房源。MVP 包含资料、资质、核验、启停用、关联查询和审计；不包含在线入驻、合同、结算、佣金或支付流程。

## 3. 用户与权限

| 角色 | 数据范围与权限 |
|---|---|
| 平台管理员 | 全部城市；新增、编辑、审核、启停用、导出、查看全部关联数据。 |
| 城市运营 | 仅授权城市；新增、编辑、提交核验、查看关联数据；无跨城市停用和资质审核权限。 |
| 销售主管 | 仅查看本团队成员所属商户的基础信息与服务范围；不可编辑、导出资质或联系人。 |
| 经纪人、客服 | 不展示本菜单。 |

联系人手机号、统一社会信用代码、资质附件下载链接均按字段权限控制。列表手机号默认脱敏；仅平台管理员和有“查看完整联系方式”权限的城市运营可查看完整值。导出继承筛选和数据权限，敏感字段按导出字段权限脱敏。

## 4. 页面入口与关联

- 菜单入口：`商户经纪 > 商户管理`。
- 前置条件：城市、区域/商圈和商户类型字典已启用；新建商户必须至少选择一个启用服务城市。
- 上游来源：商户合作资料、资质文件、城市区域配置。
- 下游去向：商户详情可跳转关联经纪人、关联楼盘、关联房源和操作日志；经纪人管理引用商户；楼盘/房源管理校验商户可发布状态；线索分配引用商户下合格经纪人的承接资格。
- 返回规则：从关联列表或详情返回时保留本页筛选、排序、分页和 URL 查询参数。

## 5. 页面结构

1. 顶部为页面标题及“新增商户”“导出”操作，操作按权限显示。
2. 查询区默认展开，包含基础筛选和“展开更多”；清空后恢复默认值。
3. 结果区显示命中数、批量启用/停用入口和商户列表，默认按更新时间倒序。
4. 详情使用右侧抽屉，依次展示基础资料、资质核验、服务范围、关联经纪人、关联楼盘、关联房源、影响处置和审计。
5. 窄屏时查询区折叠、列表横向滚动；批量操作固定在结果区顶部，不固定详情内容。

## 6. 查询与筛选

| 控件 | 数据源与规则 |
|---|---|
| 关键词 | 商户名称、内部 ID、统一社会信用代码、联系人姓名精确或模糊匹配；手机号按权限匹配，不在 URL 暴露完整值。 |
| 商户类型 | 业主、代理商、联合办公品牌，来自字典配置。 |
| 服务城市 | 已启用城市；城市运营默认其授权城市，且不可清除为跨权限范围。 |
| 核验状态 | 待提交、核验中、已通过、已驳回、已过期。 |
| 业务状态 | 草稿、启用、停用。 |
| 关联状态 | 有/无关联经纪人、关联楼盘、关联房源、上架房源。 |
| 创建/更新时间 | 北京时间范围，列表按分钟显示、导出和日志保留秒。 |

服务城市与区域/商圈联动，先选城市再加载下级项；无结果显示空状态。点击查询写入 URL，刷新或从详情返回保留；重置清除全部可编辑筛选并恢复默认排序和第 1 页。

## 7. 列表或内容展示

| 字段 | 展示与跳转规则 |
|---|---|
| 商户名称/内部 ID | 名称进入详情；内部 ID 不可变。 |
| 商户类型 | 显示字典名称；历史停用字典值保留。 |
| 联系人/联系电话 | 联系人文本；联系电话按权限完整或脱敏显示。 |
| 服务城市 | 最多展示 2 个，余项显示数量；服务城市停用时显示“已停用”。 |
| 资质核验 | 显示核验状态及最近核验时间。 |
| 关联楼盘/关联房源 | 显示总数及当前上架数，点击带商户 ID 跳转对应列表。 |
| 关联经纪人 | 显示总数、启用数、可前台展示数，点击跳转经纪人管理。 |
| 业务状态 | 草稿、启用、停用；停用行显示最近停用原因。 |
| 更新时间 | 北京时间，格式 `YYYY-MM-DD HH:mm`。 |
| 操作 | 查看、编辑、提交核验、启用、停用、查看影响；按权限和状态显示。 |

支持按创建时间、更新时间、关联房源数排序。空值显示 `-`；不存在关联数据时显示 `0`。每页默认 20 条，可切换 20/50/100 条。

## 8. 详情及表单

### 8.1 基础资料

| 字段 | 类型与校验 |
|---|---|
| 商户名称 | 必填，2-100 字；同一统一社会信用代码仅可对应一个未逻辑删除商户。 |
| 商户类型 | 必填，选择业主/代理商/联合办公品牌。 |
| 统一社会信用代码 | 企业商户必填，18 位格式校验；个体门店无法提供时须填写豁免原因并由平台管理员审核。 |
| 主联系人/联系电话 | 均必填；手机号格式校验，联系电话可在不同商户复用但保存时提示已存在记录。 |
| 服务城市/区域/商圈 | 至少一个启用服务城市；区域和商圈必须属于已选城市。 |
| 业务状态 | 新建默认为草稿，不能在编辑时直接绕过核验设为启用。 |

### 8.2 资质核验与关联

营业执照或主体证明必传，支持 JPG、PNG、PDF，单文件不超过 20 MB，至少保留有效期、附件、提交人和提交时间。核验人填写通过/驳回结论、核验备注和核验时间；到期前 30 天标记预警，到期后状态变为已过期。

详情关联页签展示关联经纪人、关联楼盘、关联房源及其当前状态。关联关系均保存不可变 ID 和生效起止时间；编辑不覆盖历史关联。保存使用版本号校验，版本冲突时阻止覆盖并提示刷新。

## 9. 核心操作与流程

### 9.1 新增、核验与启用

1. 运营创建草稿并上传资质，提交后状态为核验中。
2. 平台管理员核验通过后，商户仍需显式启用；驳回返回草稿并保留驳回原因。
3. 启用条件：资质核验已通过且未过期、至少一个服务城市启用、主联系人与联系电话完整。
4. 启用后，商户可被选择为新增/编辑楼盘和房源的供给主体；经纪人仍须各自满足其展示和承接资格才可被使用。

### 9.2 停用影响与恢复

停用前系统计算并展示影响数量：启用经纪人、可前台展示经纪人、关联楼盘、关联房源、当前上架房源、未完成线索。操作人必须填写停用原因并二次确认；存在当前上架房源或未完成线索时，必须完成以下处置计划才可提交：

1. 所有商户下经纪人立即失去“新线索分配候选人”和“前台顾问展示”资格；其二维码/联系方式入口即时失效，关联关系保留并标记为因商户停用不可用。
2. 关联楼盘和关联房源不物理删除，原审核与上架状态保留；其“商户可发布”派生条件变为否，前台不再展示，并禁止新增、编辑、重新上架或提交审核，直至商户恢复并完成复核。
3. 每条未完成线索必须在停用前转移至同城市、服务范围匹配、启用且双核验通过的经纪人；若无候选人，放入公海并记录原因“商户停用”。不得保留无负责人线索。
4. 已完成、无效、已流失线索和历史跟进保留原商户、经纪人、团队快照，不回写或删除。

恢复启用要求资质再次有效、服务城市有效，并由平台管理员复核关联房源和经纪人的资格；恢复不自动恢复前台展示、二维码、线索分配资格或房源前台可见性，须由各业务对象重新满足启用条件并显式启用。

## 10. 状态与业务规则

| 对象 | 枚举与流转 |
|---|---|
| 商户业务状态 | 草稿 -> 核验中 -> 已通过待启用 -> 启用；核验中 -> 草稿（驳回）；启用 <-> 停用。 |
| 资质核验状态 | 待提交、核验中、已通过、已驳回、已过期；已过期商户不可启用。 |
| 商户可发布 | 仅当业务状态为启用且资质为已通过、未过期时为是。 |
| 删除 | 已有关联经纪人、楼盘、房源、线索或审计记录的商户不得物理删除，仅允许停用。 |

商户名称、联系人和地址不是唯一主键；以不可变商户 ID 关联。批量停用逐条校验处置计划，展示成功/失败数量和原因。任何商户服务范围缩减均须校验关联经纪人、楼盘、房源和待分配线索；超出新范围的对象须先调整或提交例外审批。

## 11. 异常与边界

- 加载中显示骨架；查询失败显示错误提示和重试，重试保留筛选。
- 无权限访问返回无权限页；无字段权限时隐藏完整敏感值和下载入口，不以空值替代。
- 资质附件失效、无法预览或病毒扫描失败时不可提交核验，保留草稿并给出失败原因。
- 停用计算期间关联数据变化时，提交前重新计算；版本变化则要求刷新后重新选择处置。
- 下游转移任一线索失败时，整次停用不生效；批量操作逐条返回失败对象和原因。
- 已停用城市或字典值可展示历史数据，但新增/编辑不可再选择。

## 12. 数据与指标

商户主数据、资质文件元数据、城市区域、经纪人-商户关系、楼盘/房源归属、线索负责人和操作日志为数据来源。关联楼盘/关联房源数按当前关系计数；当前上架房源数仅统计房源上架且商户可发布为是的记录；可前台展示经纪人数按经纪人 PRD 的资格门槛实时计算。

列表和详情实时查询，允许缓存不超过 5 分钟；启停用、核验和转移完成后失效相关缓存。统计使用北京时间，数据明细以不可变 ID 为准。导出继承当前条件、排序和数据权限，异步生成并记录导出时间、条件、行数和脱敏策略。

## 13. 埋点、通知与审计

记录页面浏览、查询、详情打开、草稿保存、核验提交/通过/驳回、启用、停用、影响预览、导出和关联跳转。资质将到期时通知商户管理员与城市运营；核验驳回通知提交人；停用前的待转移线索通知对应销售主管。

审计字段包括操作人 ID/角色、商户 ID、操作类型、前后字段快照、核验附件标识、停用原因、影响数量、线索转移结果、时间（秒）和 IP（可选）。审计记录仅可查询，不可修改或删除。

## 14. 验收标准

| 类别 | 可测试条件 |
|---|---|
| 正常 | 核验通过且资质有效的商户可启用，并可在楼盘/房源表单中选择。 |
| 正常 | 从关联楼盘、房源、经纪人进入下游列表时携带商户 ID，返回后保留查询状态。 |
| 权限 | 城市运营不能查看未授权城市商户或下载无权限资质；普通经纪人没有菜单入口。 |
| 异常 | 有上架房源或未完成线索的商户未完成处置计划时，停用被阻止且提示具体数量。 |
| 数据 | 商户停用后，关联房源前台不可见、下属经纪人不再显示为前台顾问且不可获得新线索；历史记录保持可追溯。 |
| 数据 | 恢复商户不自动恢复房源前台可见性、顾问展示或二维码，须分别复核和启用。 |

## 15. 依赖与风险

依赖账号/角色权限、城市区域、字典配置、文件存储与病毒扫描、楼盘/房源状态、经纪人资格、线索转派及操作日志服务。风险包括旧数据缺少统一社会信用代码或资质有效期、跨城市商户范围变更造成候选人不足，以及停用高关联商户带来的前台供给下降；上线前需完成历史数据补录、资格回算和停用影响演练。

## 2. 页面概述

**业务背景**：无人承接、主动释放或被回收的可继续处理需求需要公开、可控地重新分配，不能丢失客户与历史。**页面目标**：管理在公海位置的线索，提供公平认领、主管分配、回收与冲突处理。**成功标准**：每条公海线索可解释入池原因、保护期、历史负责人和认领结果，且历史全量保留。

**MVP 范围**：入池、保护期、认领资格和限额、主管分配、批量分配、超时回收、冲突处理和历史查看。**非目标**：自动抢单、积分竞价、复杂激励和交易结算。

## 3. 用户与权限

| 角色 | 数据范围与操作 |
|---|---|
| 平台管理员 | 全部授权城市；可配置保护期、认领限额、回收阈值和处理冲突。 |
| 销售主管 | 所属团队/授权城市；可查看、分配、回收、处理冲突和查看完整手机号。 |
| 经纪人 | 服务城市内且符合保护期规则的可认领线索；可查看脱敏信息、认领本人成功后的完整信息。 |
| 客服/运营 | 授权城市只读脱敏数据，不可认领或分配。 |

文件名中的“客户”是业务导航名称；公海的可流转对象始终是附着于客户的 `lead_id`。客户档案保持存在，客户本身不进入公海、不使用销售阶段。

## 4. 页面入口与关联

- 菜单入口：客户线索 > 公海客户；也可由线索池释放、主管回收、导入待分配和工作台超时提醒进入。
- 上游：非终态线索的主动释放、满足规则的回收、未分配接入；入池时保留客户、需求、来源、楼盘/房源归因、原负责人和全部历史。
- 下游：认领或主管分配后进入我的客户；历史记录进入跟进记录；查看时可跳转客户、楼盘和房源详情。
- 直达链接、认领、分配和导出均先校验城市、团队、服务范围、在职状态和数据权限。

## 5. 页面结构

页面从上至下为：统计提示（可认领数、今日剩余额度）、筛选区、按入池时间倒序的列表、批量分配工具栏、详情抽屉和认领/冲突处理弹窗。列表默认 20 条，支持 20/50/100 条；详情固定显示入池原因、保护期、当前线索阶段和完整归属历史。

## 6. 查询与筛选

| 控件 | 规则 |
|---|---|
| 关键词 | 客户姓名、公司、线索 ID、按权限手机号。 |
| 当前线索阶段 | 第 10 节唯一枚举；已转化、已流失不会出现在公海。 |
| 入池原因 | 未分配、主动释放、SLA 回收、主管回收、导入待分配。 |
| 意向城市/区域/商圈、面积/预算 | 仅启用基础数据；金额与面积规则同线索池。 |
| 来源与归因 | 前台详情咨询、列表快速咨询、委托找房、电话咨询等，及 `building_id`/`listing_id`。 |
| 入池时间、保护期结束时间、历史负责人 | 北京时间；可筛选“可认领”“保护中”“即将可认领”。 |
| 认领状态 | 当前可认领、已被锁定处理、已分配。 |

筛选写入 URL；重置回到当前用户可见范围和默认排序。

## 7. 列表或内容展示

| 字段 | 规则 |
|---|---|
| 线索/客户 | `lead_id`、客户姓名/公司与按权限脱敏手机号；不把客户 ID 当作公海状态。 |
| 当前需求 | 城市、区域、面积、预算、入驻时间摘要。 |
| 当前线索阶段 | 仅第 10 节阶段，且均为非终态。 |
| 入池信息 | 原因、入池时间、保护期结束时间、原负责人和团队。 |
| 来源与关联 | 来源、来源参数、楼盘/房源归因和历史快照。 |
| 跟进摘要 | 最后跟进、下一动作、历史跟进数量。 |
| 操作 | 认领、主管分配、查看历史；按资格、保护期和权限显示。 |

支持按入池时间、保护期结束时间、最后跟进时间排序；空时间排末。无可见数据时显示空状态，不暴露越权记录数量。

## 8. 详情及表单

详情只读展示客户档案、当前线索需求、来源参数、关联楼盘/房源、入池与归属历史、跟进和阶段历史。手机号、附件和关联对象按字段权限展示；已下架对象保留历史快照。

| 表单 | 必填项与校验 |
|---|---|
| 入池/释放 | 入池原因必填；主动释放补充说明；仅第 10 节六个非终态可提交。 |
| 认领 | 确认当前负责人、服务城市、今日/周期认领限额和保护期资格；提交时重新校验。 |
| 主管分配 | 选择在职、服务意向城市的经纪人；批量时先逐条预校验。 |
| 回收 | 回收原因和命中 SLA 证据必填；不得回收已转化或已流失。 |
| 冲突处理 | 选择“首个成功请求为准”“主管指定负责人”或“保留原负责人”，并填写理由。 |

表单提交采用版本号校验，错误定位到字段；所有危险动作二次确认。

## 9. 核心操作与流程

1. **入池触发**：待分配线索超过配置的分配时限、负责人主动释放、已分配线索逾期且主管回收、或导入后暂未分配时，可进入公海。入池不改变线索阶段，只清空当前负责人并记录原负责人、原因和时间。
2. **保护期**：主管可配置“原负责人保护期”和“入池公开等待期”。保护期内仅原负责人或主管可处理；公开后满足资格的经纪人可认领。配置修改仅影响新入池记录，已有记录保留入池时计算出的结束时间。
3. **认领资格和限额**：经纪人须在职、服务意向城市、有客户线索菜单及认领权限，且未超过按自然日配置的成功认领上限。主管和管理员可越过限额分配但必须记录理由。
4. **冲突处理**：认领请求以服务端原子锁和版本号裁决，首个成功请求获得负责人；其他请求返回“已被认领”及可见的必要信息。若原负责人保护、权限或数据归属存在争议，锁定记录由主管处理。
5. **回收**：负责人首次跟进或下次跟进超时，主管可回收；回收后写入系统记录和站内通知。被回收线索仍保留全部跟进、来源和归因。

批量分配显示影响数量、每条成功/失败与原因；认领、分配、回收、保护期配置和冲突裁决均写入审计日志。

## 10. 状态与业务规则

### 10.1 唯一线索生命周期

本模块四个页面使用完全相同的线索生命周期：`新建（new） → 待分配（pending assignment） → 跟进中（following） → 有效商机（valid opportunity） → 带看（viewing） → 谈判（negotiating） → 已转化（converted）/已流失（lost）`。

- 新建为接入校验通过的短暂状态，随后进入待分配；分配给经纪人后进入跟进中。
- 经需求确认可进入有效商机；有实际带看记录可进入带看；有谈判说明可进入谈判。主管可补充原因后回退到跟进中或有效商机，且必须保留流转历史。
- 已转化和已流失为终态，不能分配、认领、回收或放入公海；已流失必须选择流失原因。已转化只记录转化事实，不扩展合同、支付或结算。
- 公海是归属位置，不是阶段；新建、待分配、跟进中、有效商机、带看、谈判六个非终态线索可按入池、保护、认领和分配规则流入或流出公海。

### 10.2 归属与历史

客户是由 `customer_id` 标识的持久联系人/企业档案；线索是由 `lead_id` 标识的一次需求并承载阶段和公海位置。每次入池、认领、分配、释放和回收新增一条不可变归属历史，不删除原负责人、跟进、来源、附件、楼盘/房源关联或日志。规范化手机号去重和手工合并遵循线索池规则，合并不得使历史不可追溯。

## 11. 异常与边界

- 终态、无效接入、被逻辑删除或已被认领的线索不可进入或继续显示为可认领；只读历史可在权限范围内查看。
- 认领并发、页面过期或保护期刚结束时，服务端以提交时资格和原子锁结果为准，前端刷新列表。
- 经纪人停用、服务城市变化、团队越权、超过限额或无认领权限时拒绝认领并说明原因。
- 加载、查询和导出失败可重试；批量操作部分失败不得回滚已成功的逐条处理，必须返回清单。
- 下架房源、停用城市和历史负责人离职不删除历史归因；不得将其作为新的分配候选。

## 12. 数据与指标

数据来自线索、客户、归属历史、跟进、人员组织、楼盘房源、配置和操作日志；列表实时，统计按日刷新。

| 指标 | 口径 |
|---|---|
| 当前公海线索数 | 统计时点公海位置且当前阶段为非终态的去重 `lead_id` 数。 |
| 公海认领成功率 | 统计期内成功认领数 / 发起且资格校验通过的认领请求数。 |
| 回收线索数 | 统计期内因 SLA 或主管回收进入公海的去重 `lead_id` 数。 |
| 平均公海停留时长 | 统计期内从入池到成功认领/分配的时长总和 / 已流出公海的 `lead_id` 数。 |

时间使用北京时间，金额和面积口径与线索池一致。导出继承当前条件、数据/字段权限并记录审计，不输出超出权限范围的公海记录。

## 13. 埋点、通知与审计

埋点：`lead_entered_public_pool`、`public_pool_claim_attempted`、`public_pool_claimed`、`public_pool_assigned`、`lead_recovered`、`public_pool_conflict_resolved`。通知：公开可认领时通知符合范围经纪人；成功认领通知认领人和主管；被回收通知原负责人和主管。审计记录操作人、`lead_id`、客户 ID、前后负责人、公海原因、保护期、资格/限额结果、时间到秒、IP 和处理结果。

## 14. 验收标准

- **正常**：符合条件的非终态线索可入池；保护期结束后合格经纪人认领成功并进入我的客户，完整历史保留。
- **权限**：非服务城市、超过限额、无认领权限或停用经纪人不能认领；无字段权限者仅见脱敏数据。
- **异常**：并发认领仅一个请求成功；批量分配显示逐条失败原因；已转化和已流失不可入池或认领。
- **数据**：公海不作为线索阶段；列表只使用第 10 节生命周期；入池、认领、回收与来源/跟进历史均可审计追溯。

## 15. 依赖与风险

依赖线索池、我的客户、人员团队和服务范围、城市区域、站内通知、配置中心、跟进记录、楼盘房源状态、导出和操作日志。风险：保护期、限额和回收阈值需上线前配置；没有服务端原子锁会造成重复认领；导航名称可能导致将客户档案误当作公海对象。

## 2. 页面概述

**业务背景**：销售过程需要留下可审计的沟通、到访、带看和推荐证据，供负责人执行下一动作并供主管检查质量。**页面目标**：提供跨客户的跟进记录检索、详情、受控更正、关联对象追溯及导出。**成功标准**：每条记录的作者和创建时间不可篡改，内容与关联客户/线索、楼盘/房源和阶段变更可以完整还原。

**MVP 范围**：电话、微信、到访、带看和其他渠道的记录；附件、下一跟进、受控更正、筛选、导出和审计。**非目标**：电话外呼、通话录音、IM 同步、自动质检和预约排期。

## 3. 用户与权限

经纪人可新增和查看本人负责线索的记录；销售主管可查看团队记录、审批更正和导出授权范围；管理员可查看授权城市并处理审计；运营/客服按授权范围只读脱敏内容。客户手机号、附件下载、完整内容和导出列均需字段/导出权限，且不能通过关联跳转绕过。

跟进记录关联 `customer_id` 与 `lead_id`：客户是持久联系人档案，线索是本次需求并承载生命周期；记录不定义客户阶段或第二套线索状态。

## 4. 页面入口与关联

- 菜单入口：客户线索 > 跟进记录；也可从线索池、我的客户、公海客户详情的时间线进入。
- 上游：负责人新增记录、系统生成的分配/转派/认领/回收记录、受控更正申请。
- 下游：客户与当前线索详情、关联楼盘/房源详情、工作台下次跟进提醒、操作日志。
- 前置条件：人工新增必须具有该 `lead_id` 的跟进权限；关联房源推荐时房源须已上架。返回来源页时保留来源筛选条件。

## 5. 页面结构

页面由展开筛选区、记录统计与导出操作栏、按创建时间倒序的列表、详情抽屉和更正申请弹窗组成。列表默认每页 20 条、支持 20/50/100 条；详情依次展示原始内容、更正版本、关联对象、下一动作、系统事件和审计信息。记录作者和首次创建时间固定展示且不可编辑。

## 6. 查询与筛选

| 控件 | 规则 |
|---|---|
| 关键词 | 客户姓名、公司、线索 ID、经纪人、记录内容；手机号仅按字段权限搜索和显示。 |
| 跟进人/团队 | 受本人、团队、城市数据权限约束。 |
| 当前线索阶段 | 使用第 10 节唯一枚举。 |
| 渠道 | 电话、微信、到访、带看、其他；系统记录单独筛选。 |
| 跟进结果 | 已联系、未接通、需求变化、已推荐、待下次跟进等来源于字典；结果不是生命周期状态。 |
| 关联客户/楼盘/房源 | 支持 `customer_id`、`building_id`、`listing_id`；关联对象受权限和存活状态控制。 |
| 创建、下次跟进、带看时间 | 北京时间范围；支持已逾期和今日到期。 |
| 是否含附件/推荐房源/更正 | 布尔筛选。 |

筛选、排序和分页写入 URL；清空后恢复默认时间倒序。

## 7. 列表或内容展示

| 字段 | 规则 |
|---|---|
| 记录 ID / 类型 | 不可变 ID；人工记录或系统记录。 |
| 客户与线索 | 客户姓名/公司、`lead_id`、按权限脱敏手机号和当前线索阶段。 |
| 跟进人 | 人工记录显示作者；系统记录显示系统及触发人。 |
| 渠道与结果 | 电话、微信、到访、带看、其他；结果来自字典。 |
| 内容摘要 | 截断展示，详情按权限查看完整内容；更正时显示“已更正”。 |
| 关联对象 | 楼盘/房源数量与名称；失效对象展示历史快照。 |
| 下次跟进/带看 | 时间和逾期标记；没有显示 `-`。 |
| 创建时间/操作 | 北京时间到分钟；可查看、申请更正、跳转关联对象、导出。 |

支持按创建时间、下次跟进、带看时间排序，空值排末。任何物理删除的数据均不在列表中，记录本身不允许物理删除。

## 8. 详情及表单

| 分组 | 字段、校验与行为 |
|---|---|
| 新增跟进 | 渠道必填；内容至少 5 个字；结果必填；下次跟进可选但若填写必须晚于创建时间；作者和创建时间由服务端写入。 |
| 关联对象 | 客户、线索必填；楼盘可选；推荐房源可多选但必须已上架；带看记录至少关联一个楼盘或房源。 |
| 附件 | 可选截图、沟通纪要或现场资料；单个不超过 10MB，保存文件元数据、上传人和时间，下载受权限控制。 |
| 阶段影响 | 记录可作为有效商机、带看、谈判、已转化/已流失的证据；流失必须选择原因，转化/谈判须有说明。 |
| 更正申请 | 原作者在可配置时限内或主管/管理员可申请；选择更正字段、填写原因和新内容。批准后生成新版本并保留原文、作者、首次创建时间、审批人和时间。 |

不允许直接编辑或删除原记录；更正不会改变原作者、原始创建时间和原内容。保存失败定位字段并保留草稿，版本冲突要求刷新后重试。

## 9. 核心操作与流程

1. **创建**：负责人选择渠道并提交内容、结果、关联对象、附件及下一时间；服务端写入作者、创建时间和版本，并更新线索最后跟进时间、结果摘要及工作台提醒。
2. **带看与推荐**：带看为实际发生记录，须关联楼盘或房源；推荐结果须关联至少一条已上架房源。后续下架不删除历史关联。
3. **阶段变更**：负责人基于记录证据在客户详情或同一提交中申请变更，按第 10 节校验；阶段事件以系统记录写入时间线，不能由人工文本替代。
4. **更正**：提交申请后记录仍显示原版本；主管/管理员审核通过才生成更正版本，拒绝保留申请和理由。更正不允许用于隐藏不当内容，必要时使用审计备注。
5. **导出**：具有权限的用户导出当前查询集；系统异步生成，继承数据/字段权限，敏感内容和手机号按权限脱敏，并记录导出日志。

## 10. 状态与业务规则

### 10.1 唯一线索生命周期

本模块四个页面使用完全相同的线索生命周期：`新建（new） → 待分配（pending assignment） → 跟进中（following） → 有效商机（valid opportunity） → 带看（viewing） → 谈判（negotiating） → 已转化（converted）/已流失（lost）`。

- 新建为接入校验通过的短暂状态，随后进入待分配；分配给经纪人后进入跟进中。
- 经需求确认可进入有效商机；有实际带看记录可进入带看；有谈判说明可进入谈判。主管可补充原因后回退到跟进中或有效商机，并保留阶段历史。
- 已转化和已流失为终态，不能分配、认领或放入公海；已流失必须选择流失原因。已转化只记录线索转化事实，不表示合同、支付或结算。
- 公海是归属位置，不是阶段；新建、待分配、跟进中、有效商机、带看、谈判六个非终态线索可按规则流入或移出公海。

### 10.2 记录一致性

客户由 `customer_id` 标识且不承载销售阶段；线索由 `lead_id` 标识并承载上述阶段。跟进渠道和跟进结果是记录属性，不得被映射为生命周期。系统记录与人工记录均不可物理删除；所有版本、附件、来源、楼盘/房源关联、阶段事件和作者信息均需可追溯。

## 11. 异常与边界

- 经纪人试图为非本人线索新增记录、关联越权客户或导出他人记录时拒绝并提示无权限。
- 终态线索不可新增常规跟进；主管/管理员可在审计模式下补充说明，且不得改变终态或伪造原作者/时间。
- 下架、已出租或逻辑删除房源不能用于新推荐/带看；历史记录保留对象 ID 和快照。
- 附件超限、病毒扫描失败、文件类型不允许、内容不足、时间非法或结果与关联对象不匹配时阻止提交并定位原因。
- 查询、导出、上传失败提供重试；导出异步失败可查看失败原因；并发更正以版本号阻止覆盖。

## 12. 数据与指标

数据来自跟进记录、线索、客户、楼盘房源、附件、提醒、操作日志和导出任务；列表实时，指标按日刷新。

| 指标 | 口径 |
|---|---|
| 新增跟进数 | 统计期内首次创建的人工跟进 `record_id` 数，不因更正版本重复计数。 |
| 按时跟进率 | 统计期内在下次跟进时间前完成至少一条人工跟进的应跟进 `lead_id` 数 / 应跟进 `lead_id` 总数。 |
| 带看记录数 | 统计期内渠道为带看且成功保存的去重 `record_id` 数。 |
| 推荐房源记录数 | 统计期内结果为已推荐且关联至少一条房源的去重 `record_id` 数。 |

金额、面积、时间和导出口径与线索池一致；导出保留到秒并记录查询条件、字段、数量、操作者和结果。

## 13. 埋点、通知与审计

埋点：`follow_up_created`、`follow_up_corrected_requested`、`follow_up_corrected_approved`、`follow_up_exported`、`viewing_record_created`、`listing_recommended`。下次跟进创建后提醒负责人，到期/逾期通知负责人和主管；阶段变更、附件下载、更正、导出均写审计。审计字段包括操作人、`record_id`、`lead_id`、客户 ID、操作类型、前后值、原始/更正版本、来源页面、IP、时间到秒和结果。

## 14. 验收标准

- **正常**：负责人可为本人线索写入电话、微信、到访、带看或其他记录，关联对象和下次待办正确更新；推荐仅可选已上架房源。
- **权限**：经纪人仅见本人记录；主管见团队；无导出或完整手机号权限的结果、附件和导出均按权限限制。
- **异常**：原记录不能直接编辑/删除；更正必须生成可追溯新版本；并发更正、附件失败和越权关联均不能写入错误数据。
- **数据**：记录页面不发明阶段，所有阶段仅来自第 10 节；作者、首次创建时间、系统事件、关联对象和导出操作全可审计。

## 15. 依赖与风险

依赖线索池、我的客户、公海客户、账号团队权限、楼盘房源状态、附件存储与扫描、工作台提醒、字典配置、导出任务和操作日志。风险：更正时限、审批角色和附件类型需上线配置；记录不可变性依赖服务端审计和版本存储；不能将跟进结果与线索阶段混用。

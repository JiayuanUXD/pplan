## 2. 页面概述

**业务背景**：前台详情页咨询、列表页快速咨询、委托找房和电话咨询需要被统一承接、去重和分配，避免漏跟或重复联系。

**页面目标**：提供可追溯的线索接入、校验、分配与跟进入口。**成功标准**：合格线索均可定位来源、客户、归属、时限及全量历史；超过首次响应 SLA 的线索可被识别和处理。

**MVP 范围**：接入、导入、去重、合并、单条和批量分配、转派、拒绝接入、放入公海及查看历史。**非目标**：自动派单、电话外呼或录音、IM、完整预约带看和合同结算闭环。

## 3. 用户与权限

| 角色 | 数据范围与权限 |
|---|---|
| 平台管理员 | 全部城市；可配置 SLA、查看完整手机号、分配、合并、导出。 |
| 销售主管 | 所属团队和授权城市；可分配、转派、回收、查看团队完整手机号。 |
| 经纪人 | 仅本人负责线索；可查看本人完整手机号、跟进、申请转派或释放。 |
| 客服/运营 | 授权城市；可录入、补全、导入、查看脱敏信息，不可自行分配。 |

客户手机号默认显示为 `138****1234`；完整号码和导出敏感列分别受字段权限与导出权限控制。任何查询、导出和操作均按城市、团队和本人数据权限过滤。

## 4. 页面入口与关联

- 菜单入口：客户线索 > 线索池；可从工作台待分配、超时线索进入并保留筛选条件。
- 上游：前台楼盘/房源详情咨询（保留 `building_id`、`listing_id`）、列表快速咨询、委托找房、电话咨询、客服录入、运营 xlsx 导入。
- 下游：分配后进入我的客户；放入公海后进入公海客户；新增记录及历史跳转跟进记录；已转化线索可下钻对应客户。
- 前置条件：意向城市为启用城市；候选经纪人在职且服务该城市。返回列表时保留 URL 查询条件、分页和排序。

## 5. 页面结构

从上到下依次为：标题及新增/导入/批量操作区、展开的筛选区、结果与排序栏、分页列表、右侧详情抽屉。默认按创建时间倒序，页码默认 20 条，支持 20/50/100 条；批量工具栏仅在勾选后固定显示。

详情抽屉按“客户档案、当前线索需求、来源与楼盘/房源归因、负责人与 SLA、跟进及流转历史”分组。窄屏仅作为最低限度响应适配，后台以桌面操作为主。

## 6. 查询与筛选

| 控件 | 默认值与规则 |
|---|---|
| 关键词 | 客户姓名、公司、脱敏或授权后的完整手机号、线索 ID；空值返回权限范围内结果。 |
| 来源/来源参数 | 多选来源；来源参数显示页面、活动、渠道和 `building_id`/`listing_id`，无归因显示“未关联”。 |
| 当前线索阶段 | 多选；使用第 10 节的唯一枚举。 |
| 负责人/是否待分配 | 负责人受数据权限限制；待分配对应 `pending assignment`。 |
| 意向城市、区域、商圈 | 城市联动区域和商圈；只能选择启用基础数据。 |
| 面积、预算、入驻日期 | 闭区间；最小值不得大于最大值；预算同时按数值、币种、周期和单位查询。 |
| 创建、最后跟进、下次跟进时间 | 北京时间，列表到分钟；导出与日志到秒。 |
| SLA/公海状态 | 首次跟进超时、下次跟进超时、是否在公海。 |

重置恢复页面默认值；有效筛选同步写入 URL，刷新和返回均保留。

## 8. 详情及表单

新增、编辑与导入映射使用同一字段校验：

| 分组 | 字段与校验 |
|---|---|
| 客户档案 | 姓名、规范化手机号必填；公司名称可空。客户是以规范化手机号为主要识别键的长期联系人档案，**不承载线索阶段**。 |
| 线索需求 | 意向城市、来源必填；区域/商圈可多选；面积、预算最小值不得大于最大值；工位数为正整数；入驻日期、租期、特殊要求可选。 |
| 来源归因 | 来源枚举：详情咨询、列表快速咨询、委托找房、电话咨询、客服录入、运营导入、其他；保留来源参数、`building_id`、`listing_id`，关联对象不存在时保留 ID 与历史快照。 |
| 负责人 | 经纪人新建的线索可预填本人；客服/运营录入后进入待分配；候选人必须在职且服务意向城市。 |
| 附件 | 可附咨询截图或需求文件；单个不超过 10MB，记录文件名、类型、存储键和上传人；禁止将手机号写入文件名。 |

导入仅支持 xlsx，必填列为客户姓名、手机号、意向城市、线索来源。提交前预校验，错误行提供可下载报告；版本号或更新时间不一致时阻止覆盖并提示刷新。

## 9. 核心操作与流程

1. **接入和去重**：保存前对规范化手机号（去空格、区号和分隔符后统一格式）在可配置时间窗口内查询客户及非终态线索。命中时展示客户、负责人、当前线索阶段和最近跟进时间。用户可取消、在既有客户下新建独立需求，或由管理员/主管手工合并。合并保留全部来源、归因、原 ID 映射、附件和时间线，不覆盖原作者、创建时间或操作日志。
2. **拒绝接入**：缺必填字段、非法号码、重复导入且选择不创建独立需求时拒绝创建线索；记录拒绝原因和原始导入行。拒绝结果不是客户或线索生命周期状态。
3. **分配/批量分配**：主管或管理员选择一名符合城市和在职条件的经纪人；批量操作先展示影响数量、逐条校验和二次确认，返回成功、失败及原因。分配后写系统记录并通知经纪人。
4. **转派与释放**：当前负责人申请转派；主管/管理员确认后变更负责人并保留前负责人。符合第 10 节条件的非终态线索可放入公海，需填写原因并保留保护期规则。
5. **SLA**：配置项为“首次跟进时限”和“下次跟进逾期时限”；默认首次跟进 4 小时。待分配线索在创建后提醒主管，已分配未首次跟进或下次跟进逾期时提醒负责人和主管。站内提醒，不发送短信或微信。

所有危险操作均二次确认；状态、负责人、合并、导入、导出与公海操作写入操作日志。

## 10. 状态与业务规则

### 10.1 唯一线索生命周期

本模块四个页面使用完全相同的线索生命周期：`新建（new） → 待分配（pending assignment） → 跟进中（following） → 有效商机（valid opportunity） → 带看（viewing） → 谈判（negotiating） → 已转化（converted）/已流失（lost）`。

- 新建仅表示接入校验通过但尚未完成路由；系统随即进入待分配，分配给经纪人后进入跟进中。
- 跟进中可因需求确认进入有效商机；实际到访或看房记录可进入带看；谈判进入谈判。阶段可前进，不得跳过缺少的业务证据；主管可在详情中补充原因后回退到跟进中或有效商机，系统保留流转历史。
- 已转化、已流失为终态，不能分配、认领或放入公海；已流失必须选择流失原因。已转化仅记录转化事实和时间，不扩展合同、支付或结算。
- 公海是归属位置，不是阶段。新建、待分配、跟进中、有效商机、带看、谈判六个非终态线索可在符合公海规则时转入或转出。

### 10.2 对象与一致性

客户是持久的联系人/企业档案，使用 `customer_id` 标识；线索是某次咨询或需求，使用 `lead_id` 标识并关联一个客户。客户不使用上述阶段，页面中展示的“当前线索阶段”是该客户当前处理线索的阶段。楼盘和房源通过不可变 `building_id`、`listing_id` 关联；已下架或删除的展示对象不可新增推荐，但历史归因和历史关联必须保留。

## 11. 异常与边界

- 加载中显示骨架；无结果显示空状态；查询/保存失败显示错误原因和重试入口，不清空已输入数据。
- 无菜单、操作或字段权限时隐藏入口并在直达 URL 时返回无权限页；不得通过搜索或导出绕过数据权限。
- 并发分配、合并或编辑以版本号校验，旧版本提交失败并展示最新负责人、阶段和更新时间。
- 导入存在部分错误时不写入错误行，成功行可确认后写入；重复检测失败或来源对象失效时明确列出行号和原因。
- 经纪人停用或城市不匹配时不可作为新负责人；其已有历史保留，主管必须转派未终态线索。

## 12. 数据与指标

数据源为客户、线索、楼盘、房源、经纪人、跟进记录、操作日志和导出任务。列表实时查询；经营与分析指标按日刷新。

| 指标 | 口径 |
|---|---|
| 新增线索数 | 统计期内创建且接入未被拒绝的去重后 `lead_id` 数。 |
| 待分配线索数 | 当前阶段为待分配且无负责人的非终态 `lead_id` 数。 |
| 首次跟进及时率 | 首次跟进在 SLA 内完成的已分配线索数 / 统计期内应首次跟进的已分配线索数。 |
| 有效商机数 | 统计期内首次进入有效商机的去重 `lead_id` 数。 |

金额按数值、币种、计价周期和单位保存，面积以平方米原值保存、展示最多一位小数。导出继承当前筛选、排序和数据权限，手机号按字段权限脱敏，记录操作者、条件、字段、数量和时间。

## 13. 埋点、通知与审计

埋点：`lead_created`、`lead_dedup_hit`、`lead_merged`、`lead_assigned`、`lead_transferred`、`lead_released_to_public_pool`、`lead_exported`。通知：待分配通知主管；分配通知经纪人；SLA 逾期通知经纪人与主管。审计至少记录操作者、对象类型/不可变 ID、操作类型、前后值、来源页面、时间（秒）、IP 和结果。

## 14. 验收标准

- **正常**：来自四类前台/电话来源的合格咨询均创建客户关联线索，并保留楼盘或房源归因；主管可单条和批量分配。
- **权限**：经纪人仅能查看和跟进本人线索；无完整手机号或导出权限的用户只能获得脱敏数据。
- **异常**：重复手机号命中配置窗口时必须提供新需求、合并或取消；旧版本分配不得覆盖新负责人。
- **数据**：生命周期仅出现第 10 节枚举；导出条件、数据范围和审计记录与页面一致；合并后可追溯所有原始来源和历史。

## 15. 依赖与风险

依赖账号/角色、经纪人服务城市、城市区域、线索来源和原因字典、楼盘房源主数据、文件存储、站内通知、导入导出和操作日志。风险：手机号规范化策略、去重时间窗口及合并授权需在上线配置中明确；前台来源参数缺失会降低归因质量。

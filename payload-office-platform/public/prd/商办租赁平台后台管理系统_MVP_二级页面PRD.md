## 1. 文档目标

本文档在现有 MVP PRD 框架下，补齐后台所有二级导航页面的页面级 PRD。重点不是重写业务背景，而是把每个页面拆解到可设计、可开发、可测试的粒度。

本文档覆盖以下内容：

- 页面目标与用户角色。
- 页面入口与权限范围。
- 筛选项、列表字段、详情字段、表单字段。
- 核心操作、批量操作、状态流转。
- 与前台列表页、详情页展示能力的关系。
- 异常状态、空状态、验收标准。

## 2. 前台能力对后台的映射原则

58 商办、阿里商办这类前台产品的核心体验主要集中在两个页面：

- 列表页：快速筛选、对比和发现合适房源。
- 详情页：完整判断房源真实性、区位、价格、空间条件和咨询入口。

因此后台必须为前台提供稳定、标准化、可审核的数据。

## 4. 页面 PRD

## 4.1 工作台 / 我的待办

### 页面目标

集中展示当前用户需要处理的任务，让运营、主管、经纪人无需进入多个列表即可处理高优先级事项。

### 适用角色

- 平台管理员
- 运营人员
- 销售主管
- 经纪人
- 客服

### 待办类型

| 待办类型 | 来源模块 | 处理人 | 优先级 |
|---|---|---|---|
| 超时未跟进线索 | 客户线索 | 经纪人、销售主管 | 高 |
| 待分配线索 | 线索池 | 销售主管、管理员 | 高 |
| 待审核房源 | 房源审核 | 运营人员 | 中 |
| 审核驳回房源 | 房源审核 | 提交人 | 中 |
| 房源举报待处理 | 房源举报 | 运营人员 | 高 |
| 下次跟进提醒 | 跟进记录 | 经纪人 | 中 |
| 30 天未更新房源 | 房源列表 | 运营人员、商户 | 中 |

### 筛选项

- 待办类型。
- 优先级。
- 所属城市。
- 处理状态：未处理、处理中、已处理、已忽略。
- 创建时间。
- 截止时间。

### 列表字段

| 字段 | 说明 |
|---|---|
| 待办标题 | 例如“线索超过 4 小时未跟进” |
| 关联对象 | 线索、房源、举报、楼盘 |
| 关联对象摘要 | 客户名/房源标题/楼盘名称 |
| 优先级 | 高/中/低 |
| 负责人 | 当前处理人 |
| 截止时间 | 超时任务标红 |
| 状态 | 未处理/处理中/已处理 |
| 操作 | 查看、处理、忽略 |

### 核心操作

- 查看详情：跳转关联对象详情页。
- 立即处理：根据待办类型跳转到分配、跟进、审核或举报处理弹窗。
- 标记已处理：需要系统校验关联对象状态是否已满足完成条件。
- 忽略待办：必须填写忽略原因，仅主管及以上角色可见。

### 验收标准

- 不同角色只看到自己有权限处理的待办。
- 高优先级待办默认排序在前。
- 点击待办可准确跳转到关联对象。
- 处理关联对象后，待办状态自动更新。

## 4.2 工作台 / 数据概览

### 页面目标

按角色展示平台、团队或个人维度的核心经营数据，帮助用户快速理解今日状态。

### 数据卡片

| 角色 | 数据卡片 |
|---|---|
| 管理员/运营 | 待审核房源、今日新增房源、今日新增线索、超时线索、房源举报 |
| 销售主管 | 团队新增线索、待分配线索、团队跟进数、团队有效线索、超时未跟进 |
| 经纪人 | 我的新线索、今日待跟进、已推荐客户、本月跟进数、超时未跟进 |

### 图表模块

- 近 7 天线索趋势。
- 近 7 天房源新增趋势。
- 房源状态分布。
- 线索阶段分布。

### 快捷入口

- 新增房源。
- 新增线索。
- 分配线索。
- 房源审核。
- 查看我的客户。

### 交互规则

- 点击数据卡片跳转对应列表，并自动带入筛选条件。
- 图表默认展示最近 7 天，支持切换 30 天。
- 无权限的数据卡片不展示。

### 验收标准

- 数据按当前用户数据权限统计。
- 卡片数值与对应列表筛选结果一致。
- 首屏加载不超过 2 秒。

## 4.3 楼盘管理 / 楼盘列表

### 页面目标

管理前台详情页中的楼盘基础资料，为房源录入、商圈聚合和地图展示提供标准楼盘库。

### 前台映射

楼盘列表和详情页需支撑前台展示：

- 楼盘名称、地址、区域、商圈。
- 楼宇等级、物业公司、物业费。
- 交通配套、地铁距离、周边配套。
- 楼盘图集、地图坐标。
- 楼盘下可租房源数量。

### 筛选项

- 城市、区域、商圈。
- 楼盘名称。
- 楼盘等级。
- 状态。
- 是否有上架房源。
- 更新时间。

### 列表字段

| 字段 | 说明 |
|---|---|
| 楼盘名称 | 点击进入楼盘详情 |
| 城市/区域/商圈 | 用于前台区域聚合 |
| 地址 | 支持地图定位 |
| 楼盘等级 | 甲级/超甲级/园区/服务式办公 |
| 可租房源数 | 当前已上架房源 |
| 信息完整度 | 图片、坐标、配套等完整度 |
| 状态 | 启用/停用 |
| 更新时间 | 最近编辑时间 |
| 操作 | 查看、编辑、查看房源、停用 |

### 详情/编辑字段

- 基础信息：名称、别名、城市、区域、商圈、详细地址。
- 地理信息：经纬度、地铁站、距离、地图展示名。
- 楼宇参数：楼盘等级、竣工时间、总楼层、标准层面积、电梯数。
- 费用信息：物业费、停车费、空调费。
- 图片资料：封面图、楼盘图集、公共区域、外立面。
- 配套信息：会议室、前台、停车场、餐饮、健身、咖啡等。
- SEO：标题、描述、关键词。

### 核心操作

- 新增楼盘。
- 编辑楼盘。
- 停用/启用。
- 查看楼盘下房源。
- 合并重复楼盘。

### 验收标准

- 同城市下楼盘名称不可重复。
- 停用楼盘后，该楼盘不可新增房源。
- 楼盘坐标缺失时，地图找房不可展示该楼盘点位。
- 楼盘信息修改必须写入操作日志。

## 4.4 楼盘管理 / 商圈配置

### 页面目标

维护城市、行政区、商圈、地铁站之间的关系，为前台列表筛选、SEO 聚合页和地图找房提供基础数据。

### 数据结构

```text
城市
└── 行政区
    ├── 商圈
    └── 地铁站
```

### 页面布局

- 左侧树：城市、区域、商圈层级。
- 右侧详情：当前节点基础信息和关联楼盘。
- 顶部操作：新增、编辑、启用、停用、排序。

### 字段

| 字段 | 必填 | 说明 |
|---|---:|---|
| 名称 | 是 | 城市/区域/商圈/地铁站名称 |
| 类型 | 是 | 城市、区域、商圈、地铁站 |
| 上级节点 | 否 | 城市无上级 |
| URL 标识 | 是 | 用于前台 SEO URL |
| 排序 | 否 | 前台筛选排序 |
| 简介 | 否 | 用于商圈详情页 |
| 经纬度 | 否 | 地图中心点 |
| 状态 | 是 | 启用/停用 |

### 核心操作

- 新增商圈。
- 编辑商圈。
- 绑定地铁站。
- 调整排序。
- 停用节点。
- 查看关联楼盘和房源数量。

### 验收标准

- 已被楼盘使用的商圈不可删除，只能停用。
- 停用商圈后，前台筛选不展示，但历史楼盘仍保留关联。
- URL 标识全局唯一。

## 4.5 房源管理 / 房源列表

### 页面目标

管理前台列表页和详情页展示的核心房源数据，确保房源真实、完整、可筛选、可推荐。

### 前台映射

房源列表页需要后台提供：

- 标题、封面图、楼盘、区域、商圈。
- 面积、租金、工位数、装修、可租日期。
- 近地铁、免佣、实勘、可注册、带家具等标签。
- 排序权重、更新时间、前台可见状态。

详情页需要后台提供：

- 房源图集、户型图、价格面积参数。
- 楼盘配套、交通、联系人。
- 相似房源推荐所需结构化字段。

### 筛选项

- 城市、区域、商圈、楼盘。
- 房源状态、审核状态、实勘状态。
- 房源类型、装修状态、租金单位。
- 面积区间、租金区间、工位数区间。
- 是否首页推荐、是否免佣、是否有图片。
- 更新时间、创建人、联系人。

### 列表字段

| 字段 | 说明 |
|---|---|
| 房源标题 | 点击进入详情 |
| 封面图 | 取第一张房源图 |
| 所属楼盘 | 展示楼盘名称 |
| 区域/商圈 | 来自楼盘 |
| 面积/工位 | 用于前台筛选 |
| 租金 | 单价和单位 |
| 标签 | 免佣、实勘、近地铁等 |
| 审核状态 | 待审核/通过/驳回 |
| 上架状态 | 草稿/已上架/已下架/已出租 |
| 信息完整度 | 百分比 |
| 操作 | 编辑、提交审核、上下架、标记已出租 |

### 核心操作

- 新增/编辑房源。
- 保存草稿。
- 提交审核。
- 上架/下架。
- 标记已出租。
- 设置推荐。
- 复制房源。
- 查看前台预览。

### 验收标准

- 未审核通过房源不可前台展示。
- 房源必须有关联楼盘。
- 提交审核必须满足必填字段和至少 3 张图片。
- 租金、面积、状态变更必须记录日志。

## 4.6 房源管理 / 房源审核

### 页面目标

集中处理新增房源、核心字段变更、已出租标记、举报后复核等审核任务。

### 审核来源

- 新房源提交。
- 已上架房源修改价格、面积、图片、联系人。
- 商户提交房源。
- 房源举报后复核。
- 标记已出租申请。

### 筛选项

- 审核类型。
- 城市、区域、楼盘。
- 提交人。
- 提交时间。
- 审核状态。
- 是否超时。

### 审核详情模块

- 房源基础信息。
- 价格面积信息。
- 图片和户型图。
- 楼盘与地图信息。
- 联系人和归属商户。
- 变更对比：修改前/修改后。
- 历史审核记录。

### 核心操作

- 审核通过。
- 驳回。
- 要求补充资料。
- 查看前台预览。
- 查看操作日志。

### 校验规则

- 驳回必须填写原因。
- 通过前系统校验必填项、图片数量、楼盘状态。
- 价格明显异常时提示风险，但不强制拦截。
- 审核中房源不可再次提交。

### 验收标准

- 审核通过后房源进入已上架或待上架状态，MVP 默认已上架。
- 驳回后提交人收到待办。
- 审核记录不可删除。

## 4.7 房源管理 / 房源举报

### 页面目标

处理前台用户、客服或经纪人反馈的虚假房源、价格不实、已租未下架、图片不符等问题。

### 举报类型

- 已出租未下架。
- 价格不真实。
- 图片不符。
- 地址/楼盘不符。
- 联系方式错误。
- 重复房源。
- 其他。

### 筛选项

- 举报类型。
- 处理状态。
- 房源城市/区域。
- 被举报房源。
- 举报来源。
- 举报时间。
- 处理人。

### 列表字段

| 字段 | 说明 |
|---|---|
| 举报编号 | 系统生成 |
| 房源标题 | 点击进入房源详情 |
| 举报类型 | 字典项 |
| 举报内容 | 用户填写 |
| 举报来源 | 前台/客服/经纪人 |
| 状态 | 待处理/核实中/已处理/无效 |
| 处理人 | 运营人员 |
| 操作 | 查看、处理、标记无效 |

### 核心操作

- 认领处理。
- 标记核实中。
- 下架房源。
- 要求商户/经纪人更新。
- 标记举报无效。
- 合并重复举报。

### 验收标准

- 待处理举报在工作台生成待办。
- 被多次举报的房源在房源列表显示风险标识。
- 下架处理必须写入房源日志。

## 4.8 客户线索 / 线索池

## 5. 全页面通用验收标准

1. 所有列表页支持筛选、重置、分页、排序和空状态。
2. 所有详情页支持返回来源列表并保留筛选条件。
3. 所有新增/编辑页必须校验必填字段，并定位到第一个错误字段。
4. 所有危险操作必须二次确认。
5. 所有状态变更必须写入操作日志。
6. 手机号等敏感字段按角色脱敏。
7. 页面数据必须受菜单权限、操作权限和数据权限控制。
8. 列表默认每页 20 条，支持 20/50/100 条切换。
9. 查询失败展示错误提示和重试按钮。
10. MVP 阶段不强制支持复杂自定义报表，但数据口径必须可追溯到明细列表。
